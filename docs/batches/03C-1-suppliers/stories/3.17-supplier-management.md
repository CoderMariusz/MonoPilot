# Story 3.17: Supplier Management

**Batch:** 03C-1
**Story ID:** 3.17
**Priority:** P0 (Blocker for PO)
**Effort:** 5 points

---

## User Story

**As a** Purchasing user,
**I want to** manage suppliers with defaults (currency, tax code, payment terms),
**So that** PO creation is efficient with automatic data inheritance.

---

## Acceptance Criteria

### AC-17.1: Supplier CRUD
- List with search/filter by code, name, currency, is_active
- Create modal with all fields
- Edit drawer (code immutable)
- Delete blocked if has active POs

### AC-17.2: Required Fields
- code (unique per org)
- name
- currency (PLN, EUR, USD, GBP)
- tax_code_id
- payment_terms
- lead_time_days (default 7)

### AC-17.3: Optional Fields
- contact_person, email, phone
- address, city, postal_code, country
- moq, is_active

### AC-17.4: Supplier-Product Assignments
- Products tab in supplier detail
- Assign with: is_default, supplier_product_code, unit_price, lead_time_days, moq
- Only one default supplier per product

### AC-17.5: Validation
- Unique code per org
- Valid email format
- Currency in allowed list
