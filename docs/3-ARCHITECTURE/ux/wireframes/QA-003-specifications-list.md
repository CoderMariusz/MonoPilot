## Business Rules

### Specification Lifecycle
1. **Draft** â†’ **Pending** â†’ **Active** â†’ **Expired**
2. **Draft**: Can be edited/deleted freely, not used in inspections
3. **Pending**: Awaiting QA Manager approval, cannot be edited
4. **Active**: In use for inspections, cannot be edited (must clone to new version)
5. **Expired**: No longer valid, read-only, replaced by newer version

### Version Management
- Auto-increment version on new specification (v1, v2, v3...)
- Only one Active specification per product at a time
- Activating new version auto-expires current Active version
- Expiry date = new spec effective_date - 1 day (automatic)

### Approval Workflow
- Draft â†’ Submit for Approval â†’ Pending
- Pending â†’ QA Manager approves â†’ Active (with effective/expiry dates)
- Pending â†’ QA Manager rejects â†’ back to Draft (with rejection reason)
- Approval requires QA Manager or Quality Director role

### Validation Rules
- Effective date cannot be in the past (at approval time)
- Expiry date must be after effective date (minimum 1 day)
- Cannot approve if another Active spec exists (auto-expire warning shown)
- Cannot delete specification if used in any inspections (past or scheduled)
- Min value must be less than max value (for measurement parameters)
- Critical parameters cannot be removed (only edited)

### Parameter Type Constraints
- **Measurement**: Requires target, min, max, unit (numeric validation)
- **Visual**: Requires target description, no min/max (text description)
- **Attribute**: Requires target value, no min/max (pass/fail or categorical)

## Technical Notes

### Database Schema

```sql
-- quality_specifications table
CREATE TABLE quality_specifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  product_id UUID NOT NULL REFERENCES products(id),
  version INTEGER NOT NULL DEFAULT 1,
  status VARCHAR(20) NOT NULL DEFAULT 'draft', -- draft, pending, active, expired, archived
  effective_date DATE,
  expiry_date DATE,
  approved_by UUID REFERENCES users(id),
  approved_at TIMESTAMP WITH TIME ZONE,
  created_by UUID NOT NULL REFERENCES users(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  archived_at TIMESTAMP WITH TIME ZONE,
  UNIQUE (org_id, product_id, version),
  CHECK (expiry_date > effective_date OR expiry_date IS NULL),
  CHECK (status IN ('draft', 'pending', 'active', 'expired', 'archived'))
);

-- quality_spec_parameters table
CREATE TABLE quality_spec_parameters (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  spec_id UUID NOT NULL REFERENCES quality_specifications(id) ON DELETE CASCADE,
  parameter_name VARCHAR(255) NOT NULL,
  parameter_type VARCHAR(20) NOT NULL, -- measurement, visual, attribute
  target_value VARCHAR(255),
  min_value DECIMAL(10,4),
  max_value DECIMAL(10,4),
  unit VARCHAR(50),
  test_method TEXT,
  is_critical BOOLEAN DEFAULT false,
  sequence INTEGER NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  CHECK (parameter_type IN ('measurement', 'visual', 'attribute')),
  CHECK (min_value IS NULL OR max_value IS NULL OR min_value < max_value)
);

-- Indexes
CREATE INDEX idx_specs_org_product ON quality_specifications(org_id, product_id);
CREATE INDEX idx_specs_status ON quality_specifications(status);
CREATE INDEX idx_specs_effective_date ON quality_specifications(effective_date);
CREATE INDEX idx_spec_params_spec_id ON quality_spec_parameters(spec_id);
```

### RLS Policies

```sql
-- quality_specifications RLS
CREATE POLICY "Users can view specifications in their org"
  ON quality_specifications FOR SELECT
  USING (org_id = current_setting('app.current_org_id')::uuid);

CREATE POLICY "QA users can create specifications"
  ON quality_specifications FOR INSERT
  WITH CHECK (
    org_id = current_setting('app.current_org_id')::uuid
    AND auth.jwt() ->> 'role' IN ('qa_manager', 'qa_inspector', 'quality_director')
  );

CREATE POLICY "QA users can update draft specifications"
  ON quality_specifications FOR UPDATE
  USING (
    org_id = current_setting('app.current_org_id')::uuid
    AND status = 'draft'
    AND auth.jwt() ->> 'role' IN ('qa_manager', 'qa_inspector', 'quality_director')
  );

-- quality_spec_parameters RLS (inherits from parent spec)
CREATE POLICY "Users can view parameters for their org specs"
  ON quality_spec_parameters FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM quality_specifications
      WHERE id = quality_spec_parameters.spec_id
      AND org_id = current_setting('app.current_org_id')::uuid
    )
  );
```

### Caching Strategy

- **List Cache**: `org:{orgId}:quality:specifications:list:{status}:{category}` (TTL: 5 minutes)
- **Detail Cache**: `org:{orgId}:quality:specification:{specId}` (TTL: 10 minutes)
- **Active Spec Cache**: `org:{orgId}:quality:specification:active:{productId}` (TTL: 15 minutes)
- **Invalidation**: On create, update, approve, archive, delete (purge all related cache keys)

### Query Optimization

```sql
-- Optimized list query with joins
SELECT
  s.id, s.version, s.status, s.effective_date, s.expiry_date,
  p.name AS product_name, p.code AS product_code,
  u.full_name AS approved_by_name,
  COUNT(sp.id) AS parameter_count,
  COUNT(sp.id) FILTER (WHERE sp.is_critical) AS critical_count,
  STRING_AGG(sp.parameter_name, ', ' ORDER BY sp.sequence LIMIT 10) AS parameter_preview
FROM quality_specifications s
JOIN products p ON s.product_id = p.id
LEFT JOIN users u ON s.approved_by = u.id
LEFT JOIN quality_spec_parameters sp ON s.id = sp.spec_id
WHERE s.org_id = $1
  AND s.status = ANY($2) -- Filter by status
  AND (p.name ILIKE $3 OR p.code ILIKE $3 OR sp.parameter_name ILIKE $3) -- Search
GROUP BY s.id, p.name, p.code, u.full_name
ORDER BY s.created_at DESC
LIMIT 10 OFFSET $4;
```

## Error Handling

### Common Errors

| Error Code | Message | Resolution |
|------------|---------|------------|
| SPEC_PRODUCT_REQUIRED | Product ID is required | Select a product before creating specification |
| SPEC_ACTIVE_EXISTS | Active specification already exists for this product | Current active spec will be expired when new spec is approved |
| SPEC_INVALID_DATES | Expiry date must be after effective date | Adjust dates to ensure valid range |
| SPEC_APPROVAL_UNAUTHORIZED | User does not have approval permission | Contact QA Manager or Quality Director |
| SPEC_IN_USE | Cannot delete specification referenced by inspections | Archive instead of delete |
| SPEC_PARAMETER_INVALID | Parameter min/max values are invalid | Ensure min < max for measurement parameters |
| SPEC_NOT_DRAFT | Cannot edit specification that is not in Draft status | Clone to new version to make changes |
