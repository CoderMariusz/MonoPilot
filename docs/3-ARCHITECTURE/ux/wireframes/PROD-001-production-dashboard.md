## Main Actions

### Primary
- **KPI Card Click** - Navigate to respective detail page/report
- **Auto-Refresh Toggle** - Enable/disable automatic data refresh
- **Manual Refresh Button** - Trigger immediate dashboard data refresh

### Secondary (Alert Actions)
- **Reserve Materials** - Opens material reservation modal for WO
- **Contact Warehouse** - Opens communication/notification to warehouse team
- **Log Downtime** - Opens downtime logging modal (Phase 2)
- **View WO Details** - Navigate to WO detail page
- **Update Schedule** - Opens schedule adjustment modal
- **Add Notes** - Opens notes modal for WO

### WO Table Actions ([â‹®] Menu)
- **View WO Detail** - Opens WO detail page with full info
- **Pause WO** - Pause active WO (if allow_pause_wo = true), requires reason
- **Resume WO** - Resume paused WO
- **Complete WO** - Complete WO (validates outputs registered)
- **Register Output** - Opens output registration modal (desktop) or navigate to scanner
- **Consume Materials** - Opens material consumption modal (desktop) or navigate to scanner
- **View Operations** - Opens operations timeline/tracking
- **View Consumption History** - Opens consumption log for WO
- **View Output History** - Opens output registration log for WO

### Filters
- **Alert Filters**: Toggle checkboxes for Material, Delays, Quality, Machine, Yield, OEE
- **WO Table Filters**: Line (dropdown: All, Line 1, Line 2...), Product (dropdown: All, or product list), Status (dropdown: In Progress, Paused, All)
- **WO Table Sort**: Started At (desc/asc), WO Number, Product Name, Progress %

### Export
- **Export CSV** - Downloads active WOs table to CSV with all visible columns

## States

- **Loading**: Skeleton cards + skeleton alerts + skeleton table, "Loading production dashboard data..." text
- **Success**: Populated KPI cards, alerts list, active WOs table, auto-refresh active
- **Empty**: "No active production today" message, Quick actions to start first WO, View Planning Queue link, Production Guide link
- **Error**: "Failed to load production dashboard" warning, Retry + Contact Support buttons, Quick actions still available, Auto-refresh disabled

## Business Rules

### KPI Cards
- **Orders Today**: COUNT only WOs with status='Completed' AND completed_at >= today at 00:00:00
- **Units Produced**: SUM all production_outputs.quantity WHERE created_at >= today, grouped by UoM
- **Avg Yield**: Calculate per WO as (actual_qty / planned_qty * 100), then AVG across completed WOs today
- **Yield Trend**: Compare today's avg yield vs last 7 days avg yield, show â–² if higher, â–¼ if lower
- **Active WOs**: COUNT WOs with status IN ('In Progress', 'Paused')
- **Material Shortages**: COUNT DISTINCT wo_id WHERE ANY wo_material has material_availability < 80%
- **OEE Today**: AVG(oee_percent) from oee_metrics WHERE shift_date = today (Phase 2 only)

### Alerts Panel
- **Material Shortage Alert**: Trigger when wo_material.material_availability < 80%
  - Priority: HIGH (red) if < 50%, MEDIUM (yellow) if 50-79%
  - Display: WO number, product, material name, availability %, required vs available qty
  - Actions: Reserve Materials, Contact Warehouse, View WO Details
- **Machine Down Alert**: Trigger when machine.status = 'Down' AND machine has active WO
  - Priority: HIGH (red)
  - Auto-action: Pause WO with reason "Machine Breakdown"
  - Display: Machine name, line, WO number, auto-pause status
  - Actions: Log Downtime, View Downtime History, Resume WO (when machine back up)
- **WO Delayed Alert**: Trigger when wo.scheduled_date < today AND status != 'Completed'
  - Priority: MEDIUM (yellow)
  - Display: WO number, scheduled date, current date, days delayed
  - Actions: View WO, Update Schedule, Add Notes
- **Low Yield Alert**: Trigger when (actual_output / planned_output) < 0.80 AND progress > 50%
  - Priority: MEDIUM (yellow)
  - Display: WO number, current yield %, target yield %, output qty
  - Actions: View Yield Details, Material Variance, Contact Quality
- **Quality Hold Alert**: Trigger when production_output.qa_status = 'hold' OR 'rejected'
  - Priority: MEDIUM (yellow)
  - Display: LP number, WO number, qty, QA status
  - Actions: View LP, Contact Quality, View Test Results
- **OEE Below Target Alert**: Trigger when shift OEE < target_oee_percent (Phase 2)
  - Priority: MEDIUM (yellow)
  - Display: Current OEE %, target %, availability/performance/quality breakdown
  - Actions: View OEE Dashboard, Downtime Analysis, Yield Report

### Active WOs Table

- **Pause Display**: If is_paused = true, show pause_reason and pause_duration in row details
- **Progress Bar**: Visual progress bar (0-100%) based on (actual_qty / planned_qty) * 100

### Auto-Refresh
- **Default Interval**: 30 seconds (from production_settings.dashboard_refresh_seconds)
- **Min Interval**: 5 seconds (prevent server overload)
- **Max Interval**: 300 seconds (5 minutes)
- **Toggle**: User can disable auto-refresh (persists in local storage)
- **Manual Refresh**: Always available, bypasses auto-refresh timer
- **Refresh Scope**: KPIs, Alerts, Active WOs (all dashboard data)
- **Error Handling**: If refresh fails, retry after interval, show error toast (non-blocking)

### Pagination
- **Default Page Size**: 10 WOs per page (desktop), 5 per page (mobile with "Load More")
- **Max Page Size**: 50 WOs per page
- **Sorting**: Default sort by started_at DESC (most recent first)
- **Filtering**: Combine filters with AND logic (e.g., Line 1 AND In Progress)

## Performance Notes

### Query Optimization
- **KPI Cards**: Single query with multiple COUNT/SUM aggregations (index on org_id, status, created_at, completed_at)
- **Alerts**: Materialized view or indexed query on (org_id, wo_id, material_availability, status, scheduled_date)
- **Active WOs**: Index on (org_id, status, started_at DESC), JOIN with products, lines, machines
- **Material Availability**: Pre-calculate material_availability for each wo_material (cron job or trigger)
- **Yield Calculation**: Calculate yield on-demand from production_outputs SUM vs planned_qty

### Caching Strategy
```typescript
// Redis keys
```

## Error Handling

### API Errors
- **KPIs Fetch Failed**: Show error state, allow retry, Quick actions still available
- **Alerts Fetch Failed**: Show error in Alerts panel, rest of dashboard still works
- **Active WOs Fetch Failed**: Show error in WO Table section, rest of dashboard still works

### Partial Failures
- **If KPIs fail**: Show error message in KPI section, Alerts and WO Table load normally
- **If Alerts fail**: Show error in Alerts panel, KPIs and WO Table load normally
- **If WO Table fails**: Show error in WO Table section, KPIs and Alerts load normally
- **Each section has independent error boundary**: Errors don't cascade

### Network Timeout
- **KPIs**: 5s timeout, retry once on failure
- **Alerts**: 5s timeout, retry once on failure
- **Active WOs**: 10s timeout (larger dataset), retry once on failure
- **Auto-Refresh**: If refresh fails, show toast notification, retry after next interval

### Auto-Refresh Errors
- **Error Handling**: Non-blocking error toast, auto-refresh continues after interval
- **Retry Logic**: Exponential backoff if multiple failures (30s â†’ 60s â†’ 120s)
- **Disable on Repeated Failure**: After 3 consecutive failures, disable auto-refresh, show error banner with "Retry" button
