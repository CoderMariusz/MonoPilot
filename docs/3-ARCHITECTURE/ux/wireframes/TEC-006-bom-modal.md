## Overview

Full page for creating new BOMs or editing existing ones. Includes BOM header (product, version, dates) AND BOM items (components/ingredients). This is the primary interface for defining what components make a finished product.

**Business Context:**
- BOMs define what materials/components are needed to produce a finished product
- Example: FG-001 (Honey Bread) requires RM-001 (Flour 5kg), ING-002 (Honey 1L)
- Each BOM item has quantity, scrap allowance, LP consumption mode, sequence
- BOMs support byproducts (outputs from the same process)
- Version auto-increments per product (v1 â†’ v2 â†’ v3)
- Only ONE active BOM per product at any time

**Key Concepts:**
- **Input Items**: Components consumed to make the product
- **Output Items**: Byproducts generated during production
- **License Plate (LP) Consumption**: Whether to consume whole LP or partial
- **Scrap %**: Expected material loss during production
- **Sequence**: Order of consumption in production process
- **Alternative Items**: Substitutes if primary ingredient unavailable

## Key Components

### 1. Page Header
- **Back Button**: "â† Back to BOMs" - returns to TEC-005 list
- **Save Draft**: Saves BOM with status=Draft (validates header only)
- **Save BOM**: Full validation (header + requires at least 1 item)

### 2. BOM Header Section

#### Product Selector
- **Type**: Searchable dropdown (Combobox)
- **Label**: "Product *"
- **Options**: Finished Goods (FG) and WIP products only
- **Display**: "{code} - {name}"
- **Edit Mode**: Locked (cannot change after creation)

#### Version Display
- **Type**: Read-only badge
- **Value**: Auto-incremented (v1, v2, v3...)
- **Logic**: Next available version for selected product

#### Status Selector
- **Type**: Dropdown
- **Options**: Draft, Active, Phased Out, Inactive
- **Default**: Draft for new BOMs
- **Warning**: Alert when changing Active BOM

#### Effective Dates
- **From Date**: Required, date picker
- **To Date**: Optional, null = no end date
- **Validation**: No overlap with other versions of same product

#### Output Quantity
- **Quantity**: Decimal input (max 6 decimals)
- **Unit**: Dropdown from product's UoM options
- **Purpose**: Defines batch size (e.g., 100 kg per batch)

### 3. BOM Items Table

|--------|------|-------------|

- **Total Input**: Sum of all input quantities
- **Expected Output**: From header output_qty
- **Yield %**: (Output / Input) Ã— 100

### 5. Cost Summary Panel

#### Display Modes
- **Collapsed** (default): Shows only totals and last update time
- **Expanded**: Shows detailed breakdown by material and routing costs
- **Toggle**: [â„¹ï¸ View Breakdown â–¼/â–²] button

#### Cost Breakdown Components

##### Material Costs
- **Per-item breakdown**: Shows each BOM item with:
  - Material code + name
  - Total cost for this item
  - Unit calculation: (quantity Ã— unit_cost)
- **Tree structure**: Indented with â””â”€ characters for visual hierarchy
- **Sum**: Total Material Cost at top

##### Routing Costs (if assigned)
- **Labor Cost**: Total labor from all operations
  - Shows: operation count, total minutes
- **Setup Cost**: One-time batch setup cost
- **Working Cost**: Variable cost per output unit
  - Shows: cost_per_unit Ã— output_quantity
- **Overhead**: Calculated % of labor + materials
  - Shows: percentage rate

##### Total Summary
- **Total BOM Cost**: Material + Routing costs
- **Cost per Unit**: Total / output_quantity
- **Separator**: Visual divider line before totals

#### Auto-Update Triggers
- BOM item added/removed/changed
- Material cost changed (shows warning)
- Routing changed
- Output quantity changed

#### Warnings & Tips
- **Stale data**: Shows time since last calculation if >1 hour
  - "âš ï¸ Last calculated: 2025-12-14 14:30 (2 hours ago)"
  - Provides [Recalculate Costs] button
- **No routing**: "âš ï¸ Routing not assigned. Labor costs not included."
  - Link to Advanced Settings to assign routing
- **Missing prices**: Lists materials without cost data
  - "âš ï¸ 3 materials missing cost data. [Update Costs]"
  - Shows affected items with "NO PRICE" indicator
- **Help tip**: "ğŸ’¡ Tip: Assign a routing to see labor costs included"

#### Calculation Logic
```typescript
material_cost = SUM(item.quantity Ã— material.cost_per_unit)
labor_cost = SUM(operation.labor_cost) // from routing
setup_cost = routing.setup_cost
working_cost = routing.working_cost_per_unit Ã— output_qty
overhead = (materials + labor) Ã— routing.overhead_percent / 100

total_bom_cost = material_cost + labor_cost + setup_cost + working_cost + overhead
cost_per_unit = total_bom_cost / output_qty
```

## Technical Notes

### License Plate Consumption Modes

```
PARTIAL (consume_whole_lp = false):
- System picks from LP with enough quantity
- Partial qty consumed, remaining stays on LP
- Used for: bulk ingredients, continuous materials

WHOLE LP (consume_whole_lp = true):
- System picks LPs that match required qty exactly
- Entire LP consumed at once
- Used for: pre-portioned items, packaging, discrete units
- Example: "Consume 1 box of 24 bags" not "consume 24 individual bags"
```

### Sequence Logic
```
sequence determines:
1. Order of display in BOM items table
2. Order of consumption prompts in production
3. Grouping with routing operations (if operation_seq set)

Recommended increments: 10, 20, 30... (allows inserting)
```

### Scrap Calculation
```
Gross Required = Net Required Ã— (1 + scrap_percent/100)

Example:
- Net: 50 kg flour
- Scrap: 2%
- Gross: 50 Ã— 1.02 = 51 kg ordered/picked
```

### Cost Calculation Rules

```typescript
// Material costs
for each bom_item:
  if material.cost_per_unit exists:
    item_cost = item.quantity Ã— material.cost_per_unit
  else:
    item_cost = null (flag as missing)

material_cost_total = SUM(item_cost where not null)

// Routing costs (if routing assigned)
if bom.routing_id:
  labor_cost = SUM(operation.labor_cost_per_minute Ã— operation.duration)
  setup_cost = routing.setup_cost
  working_cost = routing.working_cost_per_unit Ã— bom.output_qty
  overhead = (material_cost_total + labor_cost) Ã— routing.overhead_percent / 100
else:
  routing_costs = null

// Totals
total_bom_cost = material_cost_total + labor_cost + setup_cost + working_cost + overhead
cost_per_unit = total_bom_cost / bom.output_qty

// Staleness check
if (now - calculated_at) > 1 hour:
  show_recalculate_warning = true
```

### Cost Update Triggers
- **Immediate recalculation**:
  - BOM item quantity changed
  - Output quantity changed
  - Routing changed
- **Warning only** (manual recalc required):
  - Material price changed in products table
  - Routing operation costs changed
  - More than 1 hour since last calculation

```sql
-- Material costs
SELECT
  bi.id as item_id,
  p.code as product_code,
  p.name as product_name,
  bi.quantity,
  bi.uom,
  p.cost_per_unit as unit_cost,
  bi.quantity * COALESCE(p.cost_per_unit, 0) as total_cost,
  (p.cost_per_unit IS NOT NULL) as has_cost
FROM bom_items bi
JOIN products p ON bi.product_id = p.id
WHERE bi.bom_id = :bom_id
  AND bi.is_by_product = false;

-- Routing costs (if routing assigned)
SELECT
  r.setup_cost,
  r.working_cost_per_unit,
  r.overhead_percent,
  COUNT(ro.id) as operation_count,
  SUM(ro.duration_minutes) as total_minutes,
  SUM(ro.labor_cost_per_minute * ro.duration_minutes) as labor_cost
FROM routings r
JOIN routing_operations ro ON ro.routing_id = r.id
WHERE r.id = :routing_id;
```

- Cache result for 1 hour per BOM
- Invalidate cache on: item change, routing change, output qty change
- Return `calculated_at` timestamp for staleness check
- Flag missing prices but still calculate partial total
