   - **PO Alerts**: Overdue deliveries, pending approval >2 days, inactive suppliers
   - **WO Alerts**: Material shortages, on hold >24h, past scheduled date
   - **TO Alerts**: Overdue shipments, partial shipments >3 days
   - **Action Links**: View all alerts, resolve specific alerts, track shipments

3. **Upcoming Orders Section (PRD Section 8.4)** - Tabbed view
   - **PO Calendar Tab**: Expected deliveries by date (next 7 days)

   - **Sort**: By date, status, priority

   - **Event Details**: Entity number, description, timestamp (relative), user, quick action link
   - **Load More**: Pagination for older events (10 per page)

## Main Actions

### Primary
- **KPI Card Click** - Navigate to respective detail page (PO list, approval queue, overdue list, TO list, WO schedule, shortage items)
- **Auto-Refresh Toggle** - Enable/disable automatic data refresh
- **Manual Refresh Button** - Trigger immediate dashboard data refresh

### Secondary (Upcoming Orders)
- **Switch Tab** - Toggle between PO Calendar, WO Schedule, TO Timeline
- **View Order Detail** - Navigate to PO/WO/TO detail page
- **Receive Items** - Opens receive items modal/scanner
- **Print Order** - Generate PDF

### Alert Panel Actions
- **View All Alerts** - Navigate to alert management page
- **Resolve Alert** - Mark alert as resolved or take action
- **Track Shipment** - Opens tracking interface for TO

### Recent Activity Actions
- **View PO/WO/TO** - Navigate to entity detail page
- **Review Approval** - Opens approval modal for pending PO

### Filters
- **PO Calendar Filters**: Supplier (dropdown: All, or supplier list), Status (dropdown: All, Confirmed, Receiving)
- **WO Schedule Filters**: Production Line (dropdown: All, or line list), Shift (dropdown: All, Morning, Afternoon, Night)
- **TO Timeline Filters**: Warehouse (dropdown: All, From Warehouse, To Warehouse)

### Export
- **Export CSV** - Downloads active tab data to CSV with all visible columns

## States

- **Loading**: Skeleton cards (8 KPI cards) + skeleton alert panels + skeleton upcoming orders + skeleton activity feed, "Loading planning dashboard data..." text
- **Success**: Populated 8 KPI cards, alert panels (if alerts exist), upcoming orders tabs (if any orders in next 7 days), recent activity feed, auto-refresh active
- **Empty**: "No planning activity yet" message, Quick actions to create first PO/WO/TO, Planning Guide link
- **Error**: "Failed to load planning dashboard" warning, Retry + Contact Support buttons, Quick actions still available, Auto-refresh disabled

## Business Rules

### KPI Cards (8 Cards - PRD Section 8.2)
- **Open POs**: COUNT only POs with status NOT IN ('closed', 'cancelled')
- **Open POs Value**: SUM total_amount for open POs (currency-aware, formatted)
- **POs Pending Approval**: COUNT only POs with approval_status = 'pending', target: <5
- **Avg Approval Wait**: Calculate as AVG(CURRENT_TIMESTAMP - submitted_at) for pending approvals, display in days
- **Overdue POs**: COUNT POs where expected_delivery_date < today() AND status NOT IN ('closed', 'cancelled'), target: 0
  - Show red indicator (ğŸ”´) if count > 0
- **Open TOs**: COUNT TOs with status NOT IN ('closed', 'received')
- **WOs Scheduled Today**: COUNT WOs where scheduled_date = today(), context-dependent target
- **WOs In Progress**: COUNT WOs with status = 'in_progress', capacity-dependent target
- **Material Shortages**: COUNT products where qty_on_hand < reorder_point, target: 0
- **Supplier Lead Time Avg**: AVG(lead_time_days) for suppliers with status = 'active', target: <7 days
  - Show green indicator (ğŸŸ¢) if avg <= 7 days, yellow if > 7 days

### Alert Panels (PRD Section 8.3)
- **PO Alerts**:
  - Overdue deliveries: expected_delivery_date < today() AND status != 'closed'
  - Pending approval > 2 days: approval_status = 'pending' AND submitted_at < today() - 2 days
  - Inactive suppliers: supplier.status = 'inactive'
- **WO Alerts**:
  - Material shortages: WO on hold due to missing materials
  - On hold > 24 hours: status = 'on_hold' AND hold_timestamp < today() - 24 hours
  - Past scheduled date: scheduled_date < today() AND status NOT IN ('completed', 'closed')
- **TO Alerts**:
  - Overdue shipment: scheduled_ship_date < today() AND status != 'shipped'
  - Partial shipments > 3 days: status = 'partially_shipped' AND last_shipment_date < today() - 3 days

### Upcoming Orders Section (PRD Section 8.4)
- **PO Calendar**:
  - Filter by Days: Default 7 days (configurable in settings)
  - On-Time Indicator: expected_date >= (today + lead_time_days) â†’ "On-time" (green), else "Late" (red)
  - Delivery Status: "In Transit" (tracking exists), "Awaiting" (no tracking)
  - Sort Default: Expected Date ascending
- **WO Schedule**:
  - Filter by Days: Default 7 days
  - Filter by Line: All lines or specific line
  - Material Availability: Check stock vs BOM requirements
  - Sort Default: Scheduled Date + Time ascending
- **TO Timeline**:
  - Filter by Days: Default 7 days
  - Filter by Warehouse: All, From Warehouse, To Warehouse
  - Sort Default: Scheduled Ship Date ascending

### Recent Activity Feed
- **Event Sources**:
  - PO Created: purchase_orders WHERE created_at in last 24 hours
  - PO Approval Requested: audit_logs WHERE action = 'po_submitted'
  - PO Approved: audit_logs WHERE action = 'po_approved'
  - PO Received: purchase_receipts WHERE created_at in last 24 hours
  - WO Created: work_orders WHERE created_at in last 24 hours
  - TO Created: transfer_orders WHERE created_at in last 24 hours
  - PO Overdue: purchase_orders WHERE expected_delivery_date < today AND status != 'closed' (alert event)
- **Event Priority**: Order by timestamp DESC (most recent first)
- **Default Limit**: 10 events per page
- **Event Icons**:
  - ğŸŸ¢ Green: Created, Approved (positive events)
  - ğŸŸ¡ Yellow: Approval Requested (pending action)
  - ğŸ”µ Blue: Received (neutral event)
  - ğŸ”´ Red: Overdue (alert event)

### Auto-Refresh
- **Default Interval**: 60 seconds (from planning_settings.dashboard_refresh_seconds)
- **Min Interval**: 5 seconds (prevent server overload)
- **Max Interval**: 300 seconds (5 minutes)
- **Toggle**: User can disable auto-refresh (persists in local storage)
- **Manual Refresh**: Always available, bypasses auto-refresh timer
- **Refresh Scope**: KPIs, Alerts, Upcoming Orders, Recent Activity (all dashboard data)
- **Error Handling**: If refresh fails, retry after interval, show error toast (non-blocking)

### Pagination
- **Upcoming Orders**: Default 10 per page (desktop), 5 per page (mobile with "Load More")
- **Recent Activity**: Default 10 per page (desktop/tablet), 3 per page (mobile with "Load More")

- **KPI Cards**: Single query with multiple COUNT/SUM aggregations (index on org_id, status, expected_delivery_date, approval_status, scheduled_date, reorder_point)

- **Upcoming Orders**: Index on (org_id, scheduled_date, status), JOIN with related tables (suppliers, products, warehouses, production_lines)

## Error Handling

### API Errors
- **KPIs Fetch Failed**: Show error state in KPI section, rest of dashboard still works
- **Alerts Fetch Failed**: Show error in alert panel section, rest of dashboard still works
- **Upcoming Orders Fetch Failed**: Show error in upcoming orders section, rest of dashboard still works
- **Recent Activity Fetch Failed**: Show error in activity panel, rest of dashboard still works

### Partial Failures
- **If KPIs fail**: Show error message in KPI section, Alerts and Upcoming Orders and Activity load normally
- **If Alerts fail**: Show error in alert panel section, KPIs and Upcoming Orders and Activity load normally
- **If Upcoming Orders fail**: Show error in upcoming orders section, KPIs and Alerts and Activity load normally
- **If Recent Activity fail**: Show error in activity panel, KPIs and Alerts and Upcoming Orders load normally
- **Each section has independent error boundary**: Errors don't cascade

### Network Timeout
- **KPIs**: 5s timeout, retry once on failure
- **Alerts**: 5s timeout, retry once on failure
- **Upcoming Orders**: 5s timeout, retry once on failure
- **Recent Activity**: 5s timeout, retry once on failure
- **Auto-Refresh**: If refresh fails, show toast notification, retry after next interval

### Auto-Refresh Errors
- **Error Handling**: Non-blocking error toast, auto-refresh continues after interval
- **Retry Logic**: Exponential backoff if multiple failures (60s â†’ 120s â†’ 240s)
- **Disable on Repeated Failure**: After 3 consecutive failures, disable auto-refresh, show error banner with "Retry" button
