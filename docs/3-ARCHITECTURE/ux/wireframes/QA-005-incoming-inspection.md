## Business Rules

### Inspection Creation
1. **Auto-creation Trigger**: PO receipt can auto-create inspection if configured per product
2. **Scheduled Date**: Must be today or future date
3. **Inspector Assignment**: Optional at creation, can be QA Inspector or QA Manager role
4. **Priority**: Default is "normal", can be elevated to "high" or "critical"
5. **Multiple Line Items**: One inspection per material/line item

### Visual Inspection
1. **Temperature Check**: Mandatory for refrigerated/frozen items (storage_temp_req not "Ambient")
2. **Package Integrity**: All sub-checks (container, seal, leakage, cleanliness) required
3. **Label Verification**: Product name, lot, expiry, allergen all must be verified
4. **Quantity Variance**: >5% triggers warning, >10% requires notes
5. **Overall Visual Result**:
   - **Pass**: All checks pass, temp within range, variance <5%
   - **Fail**: Any check fails, temp out of range, critical damage
   - **Review**: Minor issues, variance 5-10%

### Test Parameters
1. **Template Selection**: Auto-selected from product config, can be changed
2. **Parameter Recording**: All parameters must have measured values
3. **Result Calculation**: Auto-calculated on blur (see QA-004 business rules)
4. **Equipment Tracking**: Required if parameter marked as equipment_required
5. **Critical Parameters**: Failures trigger QA Manager + Quality Director alerts
6. **Overall Test Result**:
   - **Pass**: All parameters pass
   - **Fail**: Any parameter fails
   - **Review**: No fails, but 1+ borderline (within 5% of limit)

### CoA Validation
1. **CoA Requirement**: Configured per supplier or per material
2. **Variance Tolerance**:
   - Quantitative parameters: <5% acceptable, 5-10% review, >10% fail
   - pH: Â±0.2 acceptable
   - Microbial: <10% variance acceptable
   - Binary (Positive/Negative): Must match exactly
3. **Missing CoA**: If required and not uploaded, inspection cannot complete
4. **CoA vs. Test Mismatch**: High variance triggers review, may require re-test

### Inspection Completion
1. **All Steps Required**: Cannot complete without all 4 steps
2. **Failed Inspection Actions**:
   - Automatic quality HOLD created on LP/lot
   - Material cannot be used in production
   - QA Manager email notification (immediate)
   - Quality Director alert if critical parameter failed
3. **Disposition Required**: Failed inspections must have disposition selected
4. **NCR Creation**: Optional checkbox, auto-populates NCR with test details
5. **Supplier Quality Issue**: Optional checkbox, creates supplier quality record
6. **Immutability**: Completed inspections cannot be edited (audit trail)
7. **Electronic Signature**: Inspector must acknowledge completion

### Hold Management
1. **Auto-hold on Failure**: Failed inspection creates hold automatically
2. **Hold Scope**: Entire LP/lot placed on hold (quantity_held = full LP quantity)
3. **Hold Reason**: Auto-populated with failed parameters
4. **Hold Release**: Requires QA Manager approval (see QA-002)
5. **Accept with Deviation**: Disposition option, requires QA Manager sign-off, hold released with notes

## API Specifications

### GET /api/quality/inspections/incoming
- **Query params**: `search`, `status`, `result`, `supplier_id`, `date_from`, `date_to`, `sort`, `order`, `page`, `limit`
- **Returns**: Paginated list with PO details, material details, test results summary, counts
- **RLS**: Filtered by org_id
- **Cache**: 1 min TTL for list

### POST /api/quality/inspections
- **Body**: `{ type: "incoming", po_id, line_item_ids: [], scheduled_date, inspector_id, priority, notes }`
- **Validation**: PO exists, line items valid, scheduled_date >= today, inspector has QA role
- **Returns**: Created inspection(s) with IDs
- **Side effects**: Creates 1 inspection per line item, status = "scheduled"

### GET /api/quality/inspections/:id
- **Returns**: Full inspection with PO details, material details, all step data, test results, CoA
- **RLS**: Org_id match
- **Cache**: 5 min TTL

### POST /api/quality/inspections/:id/start
- **Body**: None
- **Validation**: Status = "scheduled"
- **Returns**: Updated inspection with status = "in_progress", inspection_date = now
- **Side effects**: Sets started timestamp, assigns current user as inspector if unassigned

### POST /api/quality/inspections/:id/complete
- **Body**: `{ visual_data: {}, test_results: [], coa_data: {}, disposition?, disposition_notes?, create_ncr, create_supplier_issue, signature_acknowledged }`
- **Validation**: All steps complete, disposition required if failed, notes required if failed
