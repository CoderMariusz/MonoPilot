# SHIP-007: Sales Order Detail

**Module**: Shipping Management
**Feature**: Sales Order Detail View, Confirmation, Cancellation, & Inventory Allocation
**Scope**: FR-7.11 (SO Details), FR-7.13 (Confirm), FR-7.17 (Cancel)
**Status**: Ready for Review
**Last Updated**: 2025-12-15

---

## Overview

Sales Order detail view displays complete order information including header, line items, inventory allocations, status timeline, and actions for confirmation/cancellation. Handles multi-state workflows with allocation visibility and confirmation dialogs. Supports concurrent edit handling with version-based conflict resolution, comprehensive error scenarios including rate limiting and service unavailability, and print preview functionality.

## Validation Rules

### SO Confirmation
- Order status must be "draft"
- All line items must have quantities > 0
- All allocations must be fully allocated (or user must confirm partial)
- Allergen validation must pass (or user must override)
- User must have "confirm" permission
- **Version field required** in request to detect concurrent edits

### SO Cancellation
- Order status must not be "shipped" or "delivered"
- Reason must be provided (from enum)
- User must confirm cancellation checkbox
- User must have "cancel" permission
- If active pick lists exist, they must be cancelled first (future flow)
- **Notes field validation**: max_length: 500, format: string
- **Version field required** in request to detect concurrent edits

### Allocation Display
- Can view allocations if user can view SO
- Cannot modify allocations if SO is Picking/Packing/Shipped/Delivered
- Cannot adjust individual allocations if SO is in locked status

### Concurrent Edit Handling
- **Version Tracking**: Every SO has a version field (v1.0, v1.1, v1.2, etc.)
- **Conflict Detection**: On PATCH/DELETE, check if version in request matches current version
- **409 Response**: If mismatch detected, return 409 Conflict with details:
  - Current version
  - Your version
  - Last modified by (user name)
  - Last modified at (timestamp)
  - List of changes made
- **UI Resolution**: Show conflict modal with option to reload or cancel action

## Error Handling

### API Errors
- **Network Error**: "Failed to load sales order. Check your connection and try again. [Retry]"
- **404 Not Found**: "Sales Order not found. It may have been deleted. [Back to Orders]"
- **403 Forbidden**: "You don't have permission to view this order."
- **500 Server Error**: "Server error loading sales order. [Retry] [Contact Support]"

### Confirmation Errors
- **Allocation Check Failed**: "Cannot confirm: Item 'Product Name' has insufficient inventory. [View Available]"
- **Allergen Alert**: "Customer has allergen restrictions. Review and confirm override? [View Details] [Cancel]"
- **Permission Denied**: "You don't have permission to confirm this order."
- **409 Conflict (Concurrent Edit)**: Show conflict modal with detailed change information and reload/cancel options

### Cancellation Errors
- **Active Pick Lists**: "Cannot cancel. Active pick lists exist. Cancel them first. [View Pick Lists]"
- **Already Shipped**: "Cannot cancel shipped orders. Contact support if issue needs correction."
- **Permission Denied**: "You don't have permission to cancel orders."
- **409 Conflict (Concurrent Edit)**: Show conflict modal with detailed change information and reload/cancel options
- **Notes Too Long**: "Cancellation notes cannot exceed 500 characters. Currently: 650 characters. [Reduce Text]"

### Rate Limiting (429)
- **Message**: "Too Many Requests - You've made too many requests. Please slow down."
- **Details**: Show rate limit info (60 requests/minute, retry after X seconds)
- **Auto-Retry**: Countdown timer with automatic retry when available
- **Actions**: [Retry in Xs] [Go Back] [Help]

### Service Unavailable (503)
- **Message**: "Service Temporarily Unavailable - We're working to fix this. Please try again soon."
- **Details**: Show maintenance status and expected resolution time
- **Actions**: [Try Again] [Status Page] [Contact Support]
- **Status Link**: Direct link to status.monopilot.io

### Partial Failures (Reallocation)
- "Successfully reallocated 2 of 3 items. Item 'Product Name' insufficient inventory (shortage: 20 units)"
- Show what was reallocated vs. what failed
- Allow user to adjust quantities or try alternative allocation strategy
