## Main Actions

### Primary Actions
- **[+ Add Address]** - Opens add address modal
  - Validates all required fields (label, street, city, postal code, country)
  - Auto-detects postal code format based on country
  - Saves to database, refreshes address list
  - Shows toast: "Address added successfully"

- **[Edit]** - Opens edit address modal
  - Pre-fills all fields with existing data
  - Validates before save
  - Shows toast: "Address updated successfully"

- **[Delete]** - Shows confirmation, then deletes
  - If deleting default address: error message "Cannot delete the default address. Set another address as default first."
  - Shows toast: "Address deleted successfully"

- **[Set as Default]** - Shows confirmation, updates default
  - Deselects old default, sets new default
  - Updates `customer_addresses.is_default = true` for new, false for old
  - Shows toast: "Default address updated"

### Secondary Actions
- **[Cancel]** - Closes modal without saving
- **[X]** - Top-right close button, same as Cancel
- **[Retry]** (error state) - Retries fetching addresses
- **[Contact Support]** (error state) - Opens support contact form

### Dock Hours JSON Structure (Standardized Format)

```json
{
  "monday": "08:00-17:00",
  "tuesday": "08:00-17:00",
  "wednesday": "08:00-17:00",
  "thursday": "08:00-17:00",
  "friday": "08:00-17:00",
  "saturday": "closed",
  "sunday": "closed"
}
```

**Format Notes**:
- Days: Always lowercase full day names (monday, tuesday, etc.)
- Hours: 24-hour format "HH:MM-HH:MM" or "closed"
- Example with mixed hours: `"monday": "06:00-18:00", "saturday": "08:00-14:00", "sunday": "closed"`

## Validation Rules

| Field | Rules | Example |
|-------|-------|---------|
| Address Label | Required, 2-100 chars, unique per customer | "Main Warehouse" |
| Street Address | Required, 2-100 chars | "123 Industrial Blvd" |
| Street Address 2 | Optional, max 100 chars | "Suite 100" |
| City | Required, 2-50 chars, letters/spaces/hyphens | "Brooklyn" |
| State/Province | Optional (required for US/CA), 2-50 chars | "NY" |
| Postal Code | Required, format varies by country | "11201" (US), "M5H 2N2" (CA) |
| Country | Required, ISO 3166-1 code | "US", "CA", "MX" |
| Dock Hours | Optional JSON object, time format HH:MM | "08:00-17:00" |
| Notes | Optional, max 500 chars | "Fragile goods, use caution" |

### Validation Timing
- **On blur**: City/State validation (format)
- **On change**: Postal code format validation (based on selected country)
- **On submit**: All required fields, uniqueness of label per customer

### Postal Code Validation by Country
- **US**: 5 digits or 5+4 format (12345 or 12345-6789)
- **CA**: A1A 1A1 format
- **MX**: 5 digits
- **Other**: Alphanumeric, 3-20 chars

## API Endpoints

### GET Addresses
```
GET /api/shipping/customers/:customerId/addresses
Response: {
  data: [
    {
      id, label, address_line1, address_line2, city, state,
      postal_code, country, is_default, dock_hours, notes,
      created_at, updated_at
    }
  ],
  pagination: {
    total: 25,
    page: 1,
    limit: 10,
    hasMore: true
  }
}
```

### POST Create Address
```
POST /api/shipping/customers/:customerId/addresses
Body: {
  label: string (required, max 100),
  address_type: "shipping" | "billing",
  address_line1: string (required, max 100),
  address_line2?: string (max 100),
  city: string (required, max 50),
  state?: string (max 50),
  postal_code: string (required),
  country: string (required, ISO code),
  is_default?: boolean,
  dock_hours?: {
    monday: "HH:MM-HH:MM" | "closed",
    tuesday: "HH:MM-HH:MM" | "closed",
    ...
  },
  notes?: string (max 500)
}
Response: {
  id: uuid,
  label: string,
  message: "Address created successfully"
}
```

### PATCH Update Address
```
PATCH /api/shipping/customers/:customerId/addresses/:addressId
Body: { ...same fields as POST... }
Response: {
  id: uuid,
  label: string,
  message: "Address updated successfully"
}
```

### DELETE Remove Address
```
DELETE /api/shipping/customers/:customerId/addresses/:addressId
Precondition: is_default === false (cannot delete default)
Response: { message: "Address deleted successfully" }
```

### PUT Set Default
```
PUT /api/shipping/customers/:customerId/addresses/:addressId/set-default
Response: {
  id: uuid,
  is_default: true,
  message: "Address set as default successfully"
}
```
