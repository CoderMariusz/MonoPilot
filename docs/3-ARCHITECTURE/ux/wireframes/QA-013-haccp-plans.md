## Business Rules

### HACCP Plan Lifecycle
1. **Draft** â†’ **Pending** â†’ **Active** â†’ **Archived**
2. **Draft**: Can be edited/deleted freely, 7 principles must be completed before submission
3. **Pending**: Awaiting QA Manager approval, cannot be edited
4. **Active**: In effect for production, read-only, annual review required
5. **Archived**: No longer valid, read-only, replaced by newer version

### Version Management
- Auto-increment version on new plan (v1, v2, v3...)
- Only one Active plan per product at a time
- Activating new version auto-archives current Active plan
- Archive date = new plan effective_date (automatic)

### Approval Workflow
- Draft â†’ Submit for Approval â†’ Pending (requires all 7 principles complete)
- Pending â†’ QA Manager approves â†’ Active (with effective & review dates)
- Pending â†’ QA Manager rejects â†’ back to Draft (with rejection reason)
- Approval requires QA Manager or Quality Director role

### Annual Review Requirements
- All Active plans require annual review (365 days from effective date)
- Review due date automatically calculated at approval
- System sends notifications at 30, 7, and 0 days before due date
- Plan can be renewed (clone to new version) or archived
- Overdue reviews trigger alerts to QA Manager

### Validation Rules
- Effective date cannot be in the past (at approval time)
- Next review date auto-calculated = effective_date + 365 days
- Cannot delete plan if used in production (past or scheduled)
- All 7 HACCP principles must have documented evidence
- Minimum 1 hazard and 1 CCP required per plan
- Hazard severity & likelihood must be assessed
- CCP critical limits must be specific and measurable
- Cannot archive active plan without replacement

### 7 HACCP Principles (Must All Be Documented)
1. **Hazard Analysis** - Identify potential hazards (biological, chemical, physical)
2. **Determine CCPs** - Identify steps in process where hazards can be controlled
3. **Establish Critical Limits** - Set control parameters for each CCP
4. **Establish Monitoring Procedures** - Define how to monitor each CCP
5. **Establish Corrective Actions** - Define actions if CCP is out of control
6. **Establish Verification Procedures** - Define testing/audits to verify system
7. **Records & Documentation** - Define records needed for HACCP compliance

## Technical Notes

### Database Schema

```sql
-- haccp_plans table
CREATE TABLE haccp_plans (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  product_id UUID NOT NULL REFERENCES products(id),
  version INTEGER NOT NULL DEFAULT 1,
  status VARCHAR(20) NOT NULL DEFAULT 'draft', -- draft, pending, active, archived
  scope TEXT,
  effective_date DATE,
  review_due_date DATE,
  approved_by UUID REFERENCES users(id),
  approved_at TIMESTAMP WITH TIME ZONE,
  created_by UUID NOT NULL REFERENCES users(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  archived_at TIMESTAMP WITH TIME ZONE,
  UNIQUE (org_id, product_id, version),
  CHECK (status IN ('draft', 'pending', 'active', 'archived')),
  CHECK (review_due_date > effective_date OR review_due_date IS NULL)
);

-- haccp_hazards table
CREATE TABLE haccp_hazards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  plan_id UUID NOT NULL REFERENCES haccp_plans(id) ON DELETE CASCADE,
  hazard_type VARCHAR(50) NOT NULL, -- Biological, Chemical, Physical, Thermal, Allergen, Environmental, Process
  hazard_description TEXT NOT NULL,
  severity VARCHAR(10) NOT NULL, -- High, Medium, Low
  likelihood VARCHAR(10) NOT NULL, -- High, Medium, Low, Very Low
  risk_level VARCHAR(10) AS (CONCAT(severity, '-', likelihood)) STORED,
  control_measures TEXT,
  is_ccp BOOLEAN DEFAULT false,
  sequence INTEGER NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  CHECK (hazard_type IN ('Biological', 'Chemical', 'Physical', 'Thermal', 'Allergen', 'Environmental', 'Process')),
  CHECK (severity IN ('High', 'Medium', 'Low')),
  CHECK (likelihood IN ('High', 'Medium', 'Low', 'Very Low'))
);

-- haccp_ccps table (Critical Control Points)
CREATE TABLE haccp_ccps (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  plan_id UUID NOT NULL REFERENCES haccp_plans(id) ON DELETE CASCADE,
  ccp_number INTEGER NOT NULL,
  process_step VARCHAR(255) NOT NULL,
  critical_limits TEXT NOT NULL,
  monitoring_method VARCHAR(255) NOT NULL,
  monitoring_frequency VARCHAR(50) NOT NULL,
  corrective_action TEXT NOT NULL,
  verification_method TEXT,
  sequence INTEGER NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE (plan_id, ccp_number)
);

-- haccp_ccp_hazards table (Junction: links CCPs to hazards)
CREATE TABLE haccp_ccp_hazards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ccp_id UUID NOT NULL REFERENCES haccp_ccps(id) ON DELETE CASCADE,
  hazard_id UUID NOT NULL REFERENCES haccp_hazards(id) ON DELETE CASCADE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE (ccp_id, hazard_id)
);

-- haccp_monitoring_records table
CREATE TABLE haccp_monitoring_records (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  plan_id UUID NOT NULL REFERENCES haccp_plans(id),
  ccp_id UUID NOT NULL REFERENCES haccp_ccps(id),
  recorded_value VARCHAR(255) NOT NULL,
  is_compliant BOOLEAN NOT NULL,
  corrective_action_required BOOLEAN DEFAULT false,
  recorded_by UUID NOT NULL REFERENCES users(id),
  recorded_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- haccp_review_schedule table
CREATE TABLE haccp_review_schedule (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  plan_id UUID NOT NULL REFERENCES haccp_plans(id),
  review_due_date DATE NOT NULL,
  assigned_to UUID NOT NULL REFERENCES users(id),
  notification_sent_at TIMESTAMP WITH TIME ZONE,
  status VARCHAR(20) NOT NULL DEFAULT 'scheduled', -- scheduled, in_progress, completed, overdue
  completed_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_haccp_plans_org_product ON haccp_plans(org_id, product_id);
CREATE INDEX idx_haccp_plans_status ON haccp_plans(status);
CREATE INDEX idx_haccp_plans_review_due ON haccp_plans(review_due_date);
CREATE INDEX idx_haccp_hazards_plan_id ON haccp_hazards(plan_id);
CREATE INDEX idx_haccp_ccps_plan_id ON haccp_ccps(plan_id);
CREATE INDEX idx_monitoring_records_plan ON haccp_monitoring_records(plan_id, recorded_at DESC);
CREATE INDEX idx_monitoring_records_ccp ON haccp_monitoring_records(ccp_id, recorded_at DESC);
CREATE INDEX idx_review_schedule_assigned ON haccp_review_schedule(assigned_to, review_due_date);
```

### RLS Policies

```sql
-- haccp_plans RLS
CREATE POLICY "Users can view plans in their org"
  ON haccp_plans FOR SELECT
  USING (org_id = current_setting('app.current_org_id')::uuid);

CREATE POLICY "QA users can create plans"
  ON haccp_plans FOR INSERT
  WITH CHECK (
    org_id = current_setting('app.current_org_id')::uuid
    AND auth.jwt() ->> 'role' IN ('qa_manager', 'qa_inspector', 'quality_director')
  );

CREATE POLICY "QA users can update draft plans"
  ON haccp_plans FOR UPDATE
  USING (
    org_id = current_setting('app.current_org_id')::uuid
    AND status = 'draft'
    AND auth.jwt() ->> 'role' IN ('qa_manager', 'qa_inspector', 'quality_director')
  );

-- haccp_hazards RLS (inherits from parent plan)
CREATE POLICY "Users can view hazards for their org plans"
  ON haccp_hazards FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM haccp_plans
      WHERE id = haccp_hazards.plan_id
      AND org_id = current_setting('app.current_org_id')::uuid
    )
  );

-- haccp_ccps RLS (inherits from parent plan)
CREATE POLICY "Users can view ccps for their org plans"
  ON haccp_ccps FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM haccp_plans
      WHERE id = haccp_ccps.plan_id
      AND org_id = current_setting('app.current_org_id')::uuid
    )
  );
```

### Caching Strategy

- **List Cache**: `org:{orgId}:quality:haccp:plans:list:{status}:{review_status}` (TTL: 5 minutes)
- **Detail Cache**: `org:{orgId}:quality:haccp:plan:{planId}` (TTL: 10 minutes)
- **Active Plan Cache**: `org:{orgId}:quality:haccp:plan:active:{productId}` (TTL: 15 minutes)
- **Monitoring Cache**: `org:{orgId}:quality:haccp:monitoring:{planId}` (TTL: 1 minute, real-time)
- **Invalidation**: On create, update, approve, schedule review, new monitoring record, delete (purge all related cache keys)
