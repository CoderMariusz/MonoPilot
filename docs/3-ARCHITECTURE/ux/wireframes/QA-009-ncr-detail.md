### Workflow Transitions

1. **Draft â†’ Open**: Auto-transition on NCR creation, assigns owner, sets due date
2. **Open â†’ Investigation**: Auto-transition (immediate), starts investigation phase
3. **Investigation â†’ Root Cause**: Requires investigation notes, evidence, timeline (manual transition)
4. **Root Cause â†’ Corrective Action**: Requires root cause description (min 100 chars), contributing factors (manual transition)
5. **Corrective Action â†’ Verification**: Requires action plan, all action items complete (manual transition)
6. **Verification â†’ Closed**: Requires verification date, verified by, effectiveness=Pass, batch disposition, e-signature (manual transition)
7. **Closed â†’ Investigation/Root Cause/CAPA**: Reopen with justification, reassign owner, restart from selected step (QA Manager only)

### Due Date SLA

| Severity | Response Time | Due Date Calculation | Escalation |
|----------|---------------|----------------------|------------|
| Critical | 24 hours | detected_date + 24h | QA Manager + Quality Director |
| Major | 48 hours | detected_date + 48h | QA Manager |
| Minor | 7 days | detected_date + 7d | QA Team |

### Permissions by Workflow Step

| Step | Owner Can | QA Manager Can | Quality Director Can | Others Can |
|------|-----------|----------------|---------------------|-----------|
| Draft | Edit, Submit | Edit, Submit, Delete | Edit, Submit, Delete | View |
| Open | - | Assign, Change Severity | Assign, Change Severity | View |
| Investigation | Add notes, Upload evidence, Submit | Same + Reassign | Same + Reassign | View, Upload evidence |
| Root Cause | Document root cause, Submit | Same + Edit | Same + Edit | View |
| Corrective Action | Create action items, Update status | Same + Reassign items | Same + Reassign items | Update assigned items |
| Verification | - | Verify, Close, Reopen | Verify, Close, Reopen | View |
| Closed | - | Reopen | Reopen | View |

### Validation Rules

1. **Investigation â†’ Root Cause**: Investigation notes (min 50 chars), at least 1 evidence file, timeline populated
2. **Root Cause â†’ CAPA**: Root cause description (min 100 chars), at least 1 contributing factor selected
3. **CAPA â†’ Verification**: Action plan (min 100 chars), at least 1 action item, all action items status = Complete
4. **Verification â†’ Closed**: Verification date â‰¤ today, verified by = QA Manager or Quality Director, effectiveness = Pass, batch disposition selected, e-signature provided
5. **Reopen**: Justification (min 100 chars), reassign owner, select restart step

| NCR Closed | Creator, Owner, QA Team | Email | Low |
| NCR Reopened | New Owner, QA Manager | Email | High |

### Immutability Rules

- **Investigation notes**: Can edit until "Submit Investigation" - then locked
- **Root cause**: Can edit until "Submit Root Cause" - then locked
- **Action plan**: Can edit action items until all marked Complete - then locked
- **Verification**: Can edit until "Close NCR" - then locked
- **Closed NCR**: All fields read-only except Reopen action (QA Manager only)
- **Reopened NCR**: Previous workflow cycle preserved in history, new cycle created

### Attachment Rules

- **File types**: Images (JPG, PNG, PDF), Documents (PDF, DOCX, XLSX), max 10MB per file
- **Max attachments**: 50 per NCR (across all tabs)
- **Naming**: Auto-rename to `{tab}-{filename}-{timestamp}.{ext}`
- **Delete**: Can delete until step is submitted (then locked)
- **Virus scan**: All uploads scanned before storage

```sql
-- ncr_reports table
CREATE TABLE ncr_reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id UUID NOT NULL REFERENCES organizations(id),
  ncr_number VARCHAR(50) NOT NULL UNIQUE, -- NCR-YYYY-######
  title VARCHAR(255) NOT NULL,
  description TEXT NOT NULL,
  severity VARCHAR(20) NOT NULL, -- critical, major, minor
  status VARCHAR(50) NOT NULL DEFAULT 'draft', -- draft, open, investigation, root_cause, corrective_action, verification, closed
  detected_date TIMESTAMP WITH TIME ZONE NOT NULL,
  detected_by UUID NOT NULL REFERENCES users(id),
  owner_id UUID NOT NULL REFERENCES users(id),
  due_date TIMESTAMP WITH TIME ZONE NOT NULL,
  reference_type VARCHAR(50), -- inspection, production, customer_complaint, audit, other
  reference_id UUID,
  priority VARCHAR(20) NOT NULL, -- p1_critical, p2_high, p3_medium, p4_low
  immediate_containment TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  closed_at TIMESTAMP WITH TIME ZONE,
  closed_by UUID REFERENCES users(id),
  CHECK (severity IN ('critical', 'major', 'minor')),
  CHECK (status IN ('draft', 'open', 'investigation', 'root_cause', 'corrective_action', 'verification', 'closed')),
  CHECK (priority IN ('p1_critical', 'p2_high', 'p3_medium', 'p4_low'))
);

-- ncr_workflow table (workflow step history)
CREATE TABLE ncr_workflow (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ncr_id UUID NOT NULL REFERENCES ncr_reports(id) ON DELETE CASCADE,
  workflow_cycle INTEGER NOT NULL DEFAULT 1, -- Increments on reopen
  step VARCHAR(50) NOT NULL, -- draft, open, investigation, root_cause, corrective_action, verification, closed
  assigned_to UUID NOT NULL REFERENCES users(id),
  status VARCHAR(20) NOT NULL DEFAULT 'pending', -- pending, in_progress, completed
  due_date TIMESTAMP WITH TIME ZONE,
  started_at TIMESTAMP WITH TIME ZONE,
  completed_date TIMESTAMP WITH TIME ZONE,
  notes TEXT,
  investigation_notes TEXT,
  investigation_timeline JSONB, -- Array of timeline events
  five_why_analysis JSONB, -- 5 whys JSON
  root_cause_description TEXT,
  contributing_factors TEXT[], -- Array: equipment, material, method, process, people, environment
  factor_details JSONB, -- JSON with details per factor
  containment_actions TEXT[],
  action_plan TEXT,
  verification_method TEXT,
  verification_date DATE,
  verified_by UUID REFERENCES users(id),
  effectiveness_result VARCHAR(20), -- pass, fail
  verification_notes TEXT,
  batch_disposition VARCHAR(50), -- release, rework, scrap, return_to_supplier
  e_signature JSONB, -- { full_name, title, signed_at, password_verified }
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  CHECK (step IN ('draft', 'open', 'investigation', 'root_cause', 'corrective_action', 'verification', 'closed')),
  CHECK (status IN ('pending', 'in_progress', 'completed'))
);

-- ncr_action_items table
CREATE TABLE ncr_action_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ncr_id UUID NOT NULL REFERENCES ncr_reports(id) ON DELETE CASCADE,
  workflow_id UUID NOT NULL REFERENCES ncr_workflow(id) ON DELETE CASCADE,
  action_description TEXT NOT NULL,
  assigned_to UUID NOT NULL REFERENCES users(id),
  due_date DATE NOT NULL,
  status VARCHAR(20) NOT NULL DEFAULT 'not_started', -- not_started, in_progress, completed
  completion_percent INTEGER DEFAULT 0,
  completed_date DATE,
  sequence INTEGER NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  CHECK (status IN ('not_started', 'in_progress', 'completed')),
  CHECK (completion_percent BETWEEN 0 AND 100)
);

-- ncr_attachments table
CREATE TABLE ncr_attachments (
```

```sql
CREATE POLICY "Owners can update NCRs in draft/investigation"
  ON ncr_reports FOR UPDATE
  USING (
    org_id = current_setting('app.current_org_id')::uuid
    AND (owner_id = auth.uid() OR auth.jwt() ->> 'role' IN ('qa_manager', 'quality_director'))
    AND status IN ('draft', 'investigation')
  );

-- ncr_workflow RLS (inherits from ncr_reports)
CREATE POLICY "Users can view workflow for their org NCRs"
  ON ncr_workflow FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM ncr_reports
      WHERE id = ncr_workflow.ncr_id
      AND org_id = current_setting('app.current_org_id')::uuid
    )
  );

-- ncr_action_items RLS (inherits from ncr_reports)
-- ncr_attachments RLS (inherits from ncr_reports)
-- ncr_activity_log RLS (inherits from ncr_reports)
```

### Caching Strategy

- **Detail Cache**: `org:{orgId}:quality:ncr:{ncrId}` (TTL: 5 minutes)
- **Workflow Cache**: `org:{orgId}:quality:ncr:{ncrId}:workflow` (TTL: 3 minutes)
- **Activity Log Cache**: `org:{orgId}:quality:ncr:{ncrId}:activity` (TTL: 10 minutes)
- **Invalidation**: On any NCR update (status change, attachment upload, workflow transition)

### Performance Targets

- **Detail Load**: <1s including workflow, action items, attachments (50+ records)
- **Workflow Transition**: <500ms (DB transaction for status update + notification)
- **Attachment Upload**: <2s for 10MB file (including virus scan)
- **PDF Export**: <5s for complete NCR with 20+ pages
- **Activity Log**: <1s for 100+ entries with filtering
