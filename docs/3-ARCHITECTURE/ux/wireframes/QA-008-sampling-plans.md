5. **Action Buttons** - Apply to plan, Download table, Close

## Business Rules

### Sampling Plan Management
1. **Plan Name**: Must be unique within organization
2. **Product Scope**: Can apply to specific product OR product category (not both)
3. **One Active Plan**: Only one active plan per product + inspection type combination
4. **Lot Size Range**: lot_size_min must be < lot_size_max
5. **Sample Size**: sample_size must be â‰¤ lot_size_max
6. **Accept/Reject**: acceptance_number must be < rejection_number
7. **Delete Restriction**: Cannot delete plan if used by any active inspection
8. **Inactive Plan**: Can be set inactive, but existing inspections retain it

### AQL Method (ISO 2859)
1. **Sample Size Lookup**: Based on lot size range, inspection level, AQL
2. **Inspection Levels**:
   - **Normal**: Standard inspection, use for established suppliers
   - **Reduced**: 10+ consecutive lots passed, fewer samples
   - **Tightened**: 2+ failures in last 5 lots, more stringent
3. **AQL Values**:
   - **0.65**: Strictest (critical items, allergen-free)
   - **1.0-1.5**: Stringent (final products, customer-facing)
   - **2.5**: Standard (most food products)
   - **4.0**: Relaxed (low-risk packaging, supplies)
4. **Sample Size Varies**: For same plan, sample size calculated per inspection based on actual lot size
5. **ISO 2859 Table**: Hardcoded in system, 11 lot size ranges

### Random Sampling
1. **System Suggests**: Random unit identifiers from lot
2. **Stratified Option**: Sample from each production run/batch
3. **Time-based Option**: Sample every N minutes during production

### Systematic Sampling
1. **Every Nth Unit**: E.g., every 5th, every 10th
2. **Starting Point**: Random start, then systematic interval
3. **Coverage**: Ensures sampling across entire lot

### Custom Sampling
1. **Manual Entry**: User defines sample size, accept/reject numbers
2. **No Table Lookup**: Not tied to ISO 2859
3. **Validation**: Sample size â‰¤ lot size, acceptance < rejection

### Sampling Execution
1. **Plan Detection**: System auto-detects active plan matching product + inspection type + lot size
2. **Sample Identification**: Auto-generated IDs, sequential numbering
3. **Running Total**: Defects counted in real-time
4. **Early Termination**: Optionally stop if rejection limit reached before all samples inspected
5. **Accept Decision**: defects_found â‰¤ acceptance_number â†’ ACCEPT lot
6. **Reject Decision**: defects_found â‰¥ rejection_number â†’ REJECT lot
7. **Review Decision**: acceptance_number < defects_found < rejection_number â†’ REVIEW (manual QA Manager approval)
8. **Auto-hold**: Rejected lots trigger automatic quality HOLD
9. **NCR Creation**: Optional checkbox on rejection, auto-populates NCR with sampling details
10. **Immutability**: Sampling records cannot be edited once inspection completed

## API Specifications

### Sampling Plan Management

**GET /api/quality/sampling-plans**
- Query params: `search`, `inspection_type`, `status`, `sort`, `order`, `page`, `limit`
- Returns: Paginated list with product names, usage counts
- RLS: Filtered by org_id

**POST /api/quality/sampling-plans**
- Body: `{ plan_name, product_id, product_category, inspection_type, sampling_method, inspection_level, aql_level, lot_size_min, lot_size_max, sample_size, acceptance_number, rejection_number, is_active }`
- Validation: Name unique, one product scope (ID or category), min < max, sample â‰¤ lot_max, accept < reject
- Business Rule: Check for existing active plan with same product + inspection_type
- Returns: Created sampling plan with ID
- Side effects: If product_id provided, deactivate conflicting plans

**GET /api/quality/sampling-plans/:id**
- Returns: Full sampling plan with product details
- RLS: Org_id match

**PUT /api/quality/sampling-plans/:id**
- Body: Same as POST
- Validation: Name unique (excluding self), parameters valid
- Returns: Updated sampling plan
- Side effects: None (existing inspections not affected)

**DELETE /api/quality/sampling-plans/:id**
- Validation: Cannot delete if used by any active or recent inspection (30 days)
- Returns: Success message
- Side effects: Soft delete (deleted_at set)

**POST /api/quality/sampling-plans/:id/clone**
- Body: `{ new_name }`
- Validation: New name unique
- Returns: Cloned sampling plan with new ID
- Side effects: Creates new plan with same parameters

**GET /api/quality/sampling-plans/suggest**
- Query params: `product_id`, `inspection_type`, `lot_size`
- Returns: Active sampling plan matching criteria, or null
- Business Logic: Finds active plan where product matches AND inspection_type matches AND lot_size BETWEEN lot_size_min AND lot_size_max
- RLS: Org_id match

**POST /api/quality/sampling-plans/calculate-aql**
- Body: `{ lot_size, inspection_level, aql_level }`
- Returns: `{ sample_size, acceptance_number, rejection_number }` from ISO 2859 table
- No database interaction, pure calculation

### Sampling Record Management

**POST /api/quality/sampling-records**
- Body: `{ inspection_id, plan_id, sample_identifier, sample_sequence, location, result, defect_type, notes, photo_url }`
- Validation: Inspection exists, sample_sequence unique per inspection, defect_type required if result=fail
- Returns: Created sampling record
- Side effects:
  - Auto-generate sample_identifier if not provided
  - Auto-fill sampled_by from current user
  - Auto-fill sampled_at with current timestamp
  - Calculate running defect total for inspection

**GET /api/quality/sampling-records/inspection/:inspectionId**
- Returns: All sampling records for given inspection, ordered by sample_sequence
- Includes: Sample details, defect summary, accept/reject decision
- RLS: Org_id match

**POST /api/quality/sampling-records/decision**
- Body: `{ inspection_id, decision, disposition, decision_notes, ncr_created }`
- Validation: Decision matches calculated accept/reject logic, disposition required if reject
- Returns: Updated inspection with sampling decision
- Side effects:
  - If decision = reject: Create quality hold on lot
  - If decision = reject AND ncr_created = true: Create NCR with sampling details
  - If decision = reject: Send alert to QA Manager
  - If decision = review: Assign to QA Manager for approval
  - Update inspection status to sampling_complete

**GET /api/quality/sampling-records/defect-summary/:inspectionId**
- Returns: Defect types grouped with counts and percentages
- Example: `[{ defect_type: "Packaging damage", count: 3, percentage: 9.4 }]`
- RLS: Org_id match

    ]
  },
  reduced: { ... },
  tightened: { ... }

function calculateAQL(lotSize: number, inspectionLevel: string, aqlLevel: number) {
  const table = ISO_2859_TABLE[inspectionLevel];
  const range = table.ranges.find(r => lotSize >= r.min && lotSize <= r.max);
  return {
    sample_size: range.samples[aqlLevel],
    acceptance_number: range.accept[aqlLevel],
    rejection_number: range.reject[aqlLevel]
  };
