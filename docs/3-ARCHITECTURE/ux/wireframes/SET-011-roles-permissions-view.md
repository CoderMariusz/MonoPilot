## Data Structure

**Static Matrix** (hardcoded in frontend, no API call needed):

| Role Code | Display Name | Permissions Object |
|---|---|---|
| SUPER_ADMIN | Super Admin | `{ settings: 'CRUD', users: 'CRUD', ... }` |
| ADMIN | Admin | `{ settings: 'CRUD', users: 'CRUD', ... }` |
| PROD_MANAGER | Prod Mgr | `{ production: 'CRUD', planning: 'CRUD', ... }` |
| QUAL_MANAGER | Qual Mgr | `{ quality: 'CRUD', production: 'R', ... }` |
| WH_MANAGER | WH Mgr | `{ warehouse: 'CRUD', shipping: 'CRUD', ... }` |
| PROD_OPERATOR | Operator | `{ production: 'CRU', quality: 'R', ... }` |
| PICKER | Picker | `{ warehouse: 'CRU', shipping: 'CRU', inventory: 'R', ... }` |
| VIEWER | Viewer | `{ all: 'R' }` |
| GUEST | Guest | `{ technical: 'R', planning: 'R', warehouse: 'R', shipping: 'R' }` |
| API | API | `{ all: 'CRUD' except settings/users }` |

**Permission Levels**:
- `CRUD` = Create, Read, Update, Delete
- `CRU` = Create, Read, Update (no Delete)
- `R` = Read-only
- `-` = No Access

**New Roles Details**:

- **PICKER**:
  - Primary function: Warehouse picking operations
  - Can execute pick tasks (CRU on picking, packing, receiving)
  - Read-only access to inventory, products, and orders
  - Cannot modify warehouse master data or settings
  - Typical users: Warehouse floor workers, fulfillment staff

- **GUEST**:
  - Primary function: External stakeholder read-only access
  - Limited to operational data: Technical specs, Planning, Warehouse, Shipping
  - No access to Settings, Production execution, or Quality sensitive data
  - Cannot perform any write operations
  - Typical users: Suppliers, customers, auditors (limited scope)

- **API**:
  - Primary function: Service account for integrations
  - Full programmatic access to operational endpoints
  - Cannot modify Settings (organization, users, roles)
  - Used for ERP integrations, reporting systems, external platforms
  - Requires API key authentication, rate-limited
