## Business Rules

### KPI Cards (8 Cards)
- **Total LPs**: COUNT only LPs with status NOT IN ('consumed')
- **Available Qty**: SUM quantity for LPs with status='available', calculate % of total
- **Reserved Qty**: SUM quantity for LPs with status='reserved', calculate % of total
- **Blocked Qty**: SUM quantity for LPs with status='blocked' OR qa_status IN ('failed', 'quarantine'), calculate % of total
- **Expired LPs**: COUNT LPs where expiry_date < today(), target: 0
  - Show red indicator (ğŸ”´) if count > 0
- **Expiring Soon**: COUNT LPs where expiry_date BETWEEN today() AND today() + expiry_warning_days (default 7)
  - Show yellow indicator (ğŸŸ¡)
- **QA Pending**: COUNT LPs where qa_status='pending'
  - Highlight if any LPs pending >24h
- **Low Stock Items**: COUNT products where qty_on_hand < reorder_point, target: 0

### Aging Analysis
- **Age Ranges**:
  - < 7 days: "Fresh" (Green) - Optimal inventory
  - 7-30 days: "Good" (Green) - Normal rotation
  - 30-90 days: "Aging" (Yellow) - Monitor for FIFO/FEFO compliance
  - > 90 days: "Old" (Red) - Prioritize for use, potential waste risk
- **Calculation**: Age = today() - created_at
- **Display**: LP count, total quantity, % of total inventory

### Alert Panels
- **Expiry Alerts**:
  - Expired: expiry_date < today()
  - Expiring soon: expiry_date BETWEEN today() AND today() + expiry_warning_days
- **QA Alerts**:
  - Pending QA >24h: qa_status = 'pending' AND created_at < today() - 24 hours
  - Failed QA: qa_status = 'failed', status = 'blocked' or moved to quarantine location
- **Low Stock Alerts**:
  - Below reorder point: qty_on_hand < reorder_point for product
  - Show shortfall: reorder_point - available_qty

### Recent Activity Feed
- **Event Sources**:
  - LP Received: grns.created_at in last 24 hours
  - LP Moved: stock_moves WHERE move_type='transfer' in last 24 hours
  - LP Split: lp_genealogy WHERE operation_type='split' in last 24 hours
  - LP Merged: lp_genealogy WHERE operation_type='merge' in last 24 hours
  - LP Reserved: lp_reservations.created_at in last 24 hours
  - QA Status Changed: audit_logs WHERE action='qa_status_changed' in last 24 hours
  - LP Consumed: stock_moves WHERE move_type='issue' in last 24 hours
- **Event Priority**: Order by timestamp DESC (most recent first)
- **Default Limit**: 10 events per page
- **Event Icons**:
  - ğŸŸ¢ Green: LP Received, QA Passed (positive events)
  - ğŸ”µ Blue: LP Moved (neutral event)
  - ğŸŸ¢ Green: LP Split (neutral event)
  - ğŸŸ¡ Yellow: LP Reserved (pending action)
  - ğŸ”´ Red: QA Failed (alert event)

### Auto-Refresh
- **Default Interval**: 60 seconds (from warehouse_settings.dashboard_refresh_seconds)
- **Min Interval**: 5 seconds (prevent server overload)
- **Max Interval**: 300 seconds (5 minutes)
- **Toggle**: User can disable auto-refresh (persists in local storage)
- **Manual Refresh**: Always available, bypasses auto-refresh timer
- **Refresh Scope**: KPIs, Aging Analysis, Alerts, Recent Activity, Quick Search (all dashboard data)
- **Error Handling**: If refresh fails, retry after interval, show error toast (non-blocking)

### Pagination
- **Aging Analysis**: Not paginated (4 fixed rows)
- **Recent Activity**: Default 10 per page (desktop/tablet), 3 per page (mobile with "Load More")
- **Quick Inventory Search**: Default 10 per page (desktop), 5 per page (mobile with "Load More")
- **Max Page Size**: 50 items per page
- **Filtering**: Combine filters with AND logic

## Error Handling

### API Errors
- **KPIs Fetch Failed**: Show error state in KPI section, rest of dashboard still works
- **Aging Fetch Failed**: Show error in aging section, rest of dashboard still works
- **Alerts Fetch Failed**: Show error in alert panel section, rest of dashboard still works
- **Recent Activity Fetch Failed**: Show error in activity panel, rest of dashboard still works
- **Quick Search Fetch Failed**: Show error in search section, rest of dashboard still works

### Partial Failures
- **If KPIs fail**: Show error message in KPI section, Aging and Alerts and Activity and Search load normally
- **If Aging fail**: Show error in aging section, KPIs and Alerts and Activity and Search load normally
- **If Alerts fail**: Show error in alert panel section, KPIs and Aging and Activity and Search load normally
- **If Recent Activity fail**: Show error in activity panel, KPIs and Aging and Alerts and Search load normally
- **If Quick Search fail**: Show error in search section, KPIs and Aging and Alerts and Activity load normally
- **Each section has independent error boundary**: Errors don't cascade

### Network Timeout
- **KPIs**: 5s timeout, retry once on failure
- **Aging**: 5s timeout, retry once on failure
- **Alerts**: 5s timeout, retry once on failure
- **Recent Activity**: 5s timeout, retry once on failure
- **Quick Search**: 5s timeout, retry once on failure
- **Auto-Refresh**: If refresh fails, show toast notification, retry after next interval

### Auto-Refresh Errors
- **Error Handling**: Non-blocking error toast, auto-refresh continues after interval
- **Retry Logic**: Exponential backoff if multiple failures (60s â†’ 120s â†’ 240s)
- **Disable on Repeated Failure**: After 3 consecutive failures, disable auto-refresh, show error banner with "Retry" button
