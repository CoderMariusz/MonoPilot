## 1. Feature Overview

The Shipping Dashboard provides real-time visibility into all shipping operations, including sales orders, pick lists, shipments, and KPIs. It serves as the central command center for warehouse managers and shipping supervisors to monitor workflow status, identify bottlenecks, and prioritize actions.

**Key Goals:**
- Real-time overview of shipping operations
- Quick access to pending tasks and alerts
- Performance metrics tracking
- Activity feed for audit trail
- One-click actions for common workflows

**Related Features:**
- SHIP-001: Customer List
- SHIP-007: Sales Order List
- SHIP-013: Pick List
- SHIP-019: Shipments List
- FR-7.66: Shipping Dashboard (PRD)

## 4. Key Metrics & KPI Definitions

### 4.1 Orders Pending Confirmation

**Metric:** Count of sales orders with status = 'draft'
**Target:** < 5 per day
**Calculation:** `COUNT(sales_orders WHERE status = 'draft' AND created_date = TODAY)`
**Status Indicator:**
- Green: 0-3
- Yellow: 4-6
- Red: 7+

**Action:** Review and confirm orders

---

### 4.2 Pick Lists In Progress

**Metric:** Count of pick lists with status = 'assigned' OR 'in_progress'
**Target:** 5-12 concurrent (depends on warehouse size)
**Calculation:** `COUNT(pick_lists WHERE status IN ('assigned', 'in_progress'))`
**Status Indicator:**
- Green: Assigned to 3+ pickers, progressing
- Yellow: Only 1-2 pickers, slow progress
- Red: 0 pickers assigned, backlog building

**Action:** Assign additional pickers if backlog

---

### 4.3 Shipments Today

**Metric:** Count of shipments with status = 'shipped' on current date
**Target:** Varies by company (10-50+ per day)
**Calculation:** `COUNT(shipments WHERE status = 'shipped' AND shipped_at::date = TODAY)`
**Status Indicator:**
- Green: > target
- Yellow: 80-100% of target
- Red: < 80% of target

**Action:** Accelerate packing/shipping

---

### 4.4 On-Time Delivery Rate

**Metric:** Percentage of orders shipped by promised date
**Target:** > 95%
**Calculation:**
```
COUNT(shipments WHERE shipped_at <= promised_ship_date)
/
COUNT(shipments WHERE status = 'shipped')
* 100
```
**Time Window:** Last 30 days
**Status Indicator:**
- Green: > 95%
- Yellow: 90-95%
- Red: < 90%

**Trend:** Show 30-day moving average, compare week-over-week

---

### 4.5 Backorders

**Metric:** Count of sales orders with status = 'partial' (incomplete allocation)
**Target:** Minimize (ideally 0, accept 1-2)
**Calculation:**
```
COUNT(sales_orders
  WHERE status = 'partial'
  OR EXISTS (
    SELECT 1 FROM sales_order_lines sol
    WHERE sol.sales_order_id = sales_orders.id
    AND sol.quantity_allocated < sol.quantity_ordered
  )
)
```
**Urgency Levels:**
- Urgent: No estimated ship date, customer waiting
- Standard: Scheduled for restock, waiting on incoming PO
- Future: Long lead time, no pressure

**Action:** Check available inventory, contact supplier, expedite PO

---

### 4.6 Avg Pick Time

**Metric:** Average duration from pick start to completion
**Target:** < 60 min
**Calculation:**
```
AVG(completed_at - started_at)
FROM pick_lists
WHERE completed_at IS NOT NULL
  AND completed_at::date = TODAY
```
**Status Indicator:**
- Green: < 50 min
- Yellow: 50-70 min
- Red: > 70 min

**Factors:** Number of items per pick, warehouse layout, picker efficiency

---

### 4.7 Cartons Packed Today

**Metric:** Count of shipment boxes created today
**Target:** Varies (typically 50-200+ per day)
**Calculation:** `COUNT(shipment_boxes WHERE created_at::date = TODAY)`
**Productivity:** Track boxes/hour per station

**Action:** Monitor packing station productivity

---

### 4.8 Customer Orders Today

**Metric:** Count of unique customers with shipments today
**Target:** Varies
**Calculation:**
```
COUNT(DISTINCT customer_id)
FROM shipments
WHERE shipped_at::date = TODAY
```
**Insight:** Customer concentration, top customers today

## 8. Modal Focus Management

### 8.1 Alert Action Modals

**Modal Types:**
1. Assign Picker Modal (Incomplete Picks Alert)
2. Reschedule Dock Modal (Dock Capacity Alert)
3. Create PO Modal (Low Stock Alert)
4. Track Shipment Modal (Late Shipments Alert)

### 8.2 Focus Management Specifications

**Opening Modal:**
- Focus immediately moves to first focusable element (e.g., input field or button)
- Store reference to trigger element (the alert link/button that opened modal)
- `aria-modal="true"` on modal container
- `role="dialog"` on modal element
- Modal backdrop: `aria-hidden="true"` to exclude from keyboard nav

**Inside Modal:**
- Implement focus trap: Tab key cycles through focusable elements within modal
- When Tab key pressed on last element, focus moves to first element
- When Shift+Tab pressed on first element, focus moves to last element
- Escape key closes modal immediately

**Code Implementation:**
```typescript
// Focus trap example
const focusableElements = modalRef.querySelectorAll(
  'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
);
const firstElement = focusableElements[0];
const lastElement = focusableElements[focusableElements.length - 1];

document.addEventListener('keydown', (e) => {
  if (e.key === 'Escape') {
    closeModal();
    returnFocusToTrigger();
  }

  if (e.key === 'Tab') {
    if (e.shiftKey) {
      if (document.activeElement === firstElement) {
        e.preventDefault();
        lastElement.focus();
      }
    } else {
      if (document.activeElement === lastElement) {
        e.preventDefault();
        firstElement.focus();
      }
    }
  }
});
```

**Closing Modal:**
- Escape key: Close modal, return focus to trigger element
- Cancel/Close button: Close modal, return focus to trigger element
- Submit/Action button: Execute action, close modal, return focus to trigger element
- Outside click: Close modal, return focus to trigger element

**Return Focus Implementation:**
```typescript
const triggerElement = document.activeElement; // Store before opening

// On modal close:
triggerElement.focus();
```

**Accessibility Features:**
- `aria-label="Close dialog"` on X button
- `aria-describedby="modal-description"` on modal (links to descriptive text)
- Announce modal opening to screen readers: `aria-live="polite"` on modal
- Announce modal closing and action result to screen readers
