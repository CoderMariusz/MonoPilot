## Validation Rules

### LP Validation

| Rule | Condition | Action |
|------|-----------|--------|
| **LP Exists** | LP not found | Block with error "License plate not found" |
| **Product Match** | LP product != SO line product | Block with error "Product mismatch" |
| **Allocated to SO** | LP not allocated to this SO | Block with error "Item not allocated to this order" |
| **Quantity Valid** | qty_to_pack > (ordered_qty - packed_qty) | Block with error "Exceeds ordered quantity" |
| **Quantity > 0** | qty_to_pack <= 0 | Block with error "Quantity must be > 0" |
| **Capacity Check** | current_weight + item_weight > max_weight | Show capacity error, suggest alternate carton |

### Weight Verification

| Rule | Condition | Action |
|------|-----------|--------|
| **Variance Threshold** | variance > 10% | Alert user, allow with override |
| **Variance Warning** | variance > 5% | Warn user, log variance, allow to proceed |
| **Weight Within Max** | actual_weight > max_weight | Block with error "Exceeds carton capacity" |
| **Weight Recorded** | weight_actual is null | Block with error "Weight must be entered" |

### Carton Validation

| Rule | Condition | Action |
|------|-----------|--------|
| **Carton Type Valid** | carton_type not found | Block with error "Invalid carton type" |
| **Max Weight Set** | carton_type.max_weight is null | Block, config error |
| **At Least One Item** | items_in_carton = 0 | Block with error "Carton must have at least 1 item" |
| **Dimensions Set** | dimensions are null | Block with error "Dimensions required" |

## Business Rules

### Packing Workflow

1. **Start Packing**: Select sales order â†’ System creates shipment with status 'packing'
2. **Create Carton**: Select carton type (Standard, Large, Small, Custom)
3. **Scan LP**: Validate LP, check allocation, verify capacity
4. **Add to Carton**: Record shipment_box_contents, update weight
5. **Weight Verification**: Manual scale or system estimate, validate variance
6. **SSCC Generation**: Create unique SSCC per carton (GS1 standard)
7. **Print Label**: Zebra ZPL or PDF format
8. **Create New Carton**: If multi-carton shipment, repeat steps 2-7
9. **Complete Shipment**: All items packed, set shipment.status = 'packed'
10. **Generate Manifest**: Packing slip + BOL for carrier

### Weight Calculation

```
item_weight = product.weight_per_unit * quantity_to_pack
carton_weight = SUM(item_weights)
capacity_remaining = carton_type.max_weight - carton_weight
```

### Capacity Rules

1. **Hard Limit**: Cannot exceed carton_type.max_weight by any amount
2. **Warning**: Show warning at 70% capacity
3. **Remaining Display**: Show "X kg remaining" in real-time
4. **Overflow Handling**: Reject item, suggest larger carton, or split quantity

### SSCC Generation

**Format**: (00) SSCC (18 digits)

**Structure**:
- Extension Digit (1): Fixed '0'
- GS1 Company Prefix (7-10): From org configuration
- Serial Reference (8-6): Unique sequential number per carton
- Check Digit (1): Calculated per GS1 standard

**Example**: `000123456789012345`

### Offline Mode Rules

1. **Storage**: IndexedDB for offline queue (max 50 cartons)
2. **SSCC Generation**: Deferred until online (use temp ID locally)
3. **Queue Order**: FIFO (First-In, First-Out)
4. **Auto-Sync**: When connection restored, sync cartons in order
5. **SSCC Print**: After sync, print all pending SSCC labels
6. **TTL**: Queued cartons expire after 24h (warn user)

### Network Retry Policy

**Trigger**: Scale connection timeout or network error during weight verification

**Retry Strategy**: Exponential backoff with 3 maximum attempts
- **Attempt 1**: Retry after 0.5 seconds (immediate fast retry)
- **Attempt 2**: Retry after 2.0 seconds (medium delay)
- **Attempt 3**: Retry after 5.0 seconds (longer delay)
- **After 3 failures**: Show "Scale Connection Error" with fallback options (manual entry, calibration, system estimate)

**Backoff Formula**: `delay = base_delay * (2 ^ attempt_number)`
- Attempt 1: 0.5 * 2^0 = 0.5s
- Attempt 2: 0.5 * 2^2 = 2.0s
- Attempt 3: 0.5 * 2^4 = 8.0s (capped at 5.0s)

**Escalation**: If all retries fail, display scale connection error state with:
- Troubleshooting steps (power, reconnect, physical check)
- Manual weight entry form
- System estimate option (unverified)
- Scale calibration option
- Supervisor call option

## Error Handling

### Network Errors

| Error | Trigger | Action |
|-------|---------|--------|
| **Connection Lost** | Network offline | Switch to offline mode, queue cartons |
| **Timeout (>5s)** | No response from server | Retry once, then offline mode |
| **500 Server Error** | Server failure | Show error, retry button, offline fallback |
| **401 Unauthorized** | Session expired | Force logout, redirect to login |
| **403 Forbidden** | Permission denied | Show error "No permission for this warehouse" |

### Scale Errors

| Error | Trigger | Recovery |
|-------|---------|----------|
| **Scale Disconnected** | USB/BLE connection lost | Retry (exponential backoff), troubleshoot, manual entry |
| **Scale Timeout** | No response >5s | Trigger exponential retry (0.5s, 2s, 5s), then fallback |
| **Invalid Reading** | Scale returns invalid data | Log error, ask user to verify, retry |
| **Calibration Needed** | Scale reading unstable | Suggest calibration workflow |
| **Out of Range** | Weight > scale max | Error, suggest manual entry |

### Validation Errors

| Error | Trigger | User Action |
|-------|---------|-------------|
| **LP_NOT_FOUND** | License plate doesn't exist | Scan again or enter manually |
| **PRODUCT_MISMATCH** | LP product != SO line product | Scan correct LP or search |
| **NOT_ALLOCATED** | LP not allocated to this SO | Report issue or contact supervisor |
| **QUANTITY_EXCEEDED** | qty > allocated qty | Reduce quantity or split |
| **CAPACITY_EXCEEDED** | weight > max weight | Use larger carton or split |
| **WEIGHT_VARIANCE** | variance > 10% | Accept with override or reopen |

### Scan Errors

| Error | Trigger | Recovery |
|-------|---------|----------|
| **Camera Permission Denied** | User denied camera access | Prompt to enable in settings |
| **Scanner Hardware Failure** | External scanner disconnected | Show error, prompt to reconnect |
| **Barcode Unreadable** | Poor quality barcode | Retry scan, suggest manual entry |
| **Duplicate Scan** | Same barcode scanned twice in 2s | Ignore, no action |
| **Scan Timeout** | No scan within 30s | Return to scan prompt, clear state |

### Offline Queue Errors

| Error | Trigger | Recovery |
|-------|---------|----------|
| **Queue Full** | 50 cartons in queue | Block new cartons, prompt to sync |
| **Sync Conflict** | Server validation fails on sync | Flag carton, show conflict resolution UI |
| **TTL Expired** | Queued carton >24h old | Remove from queue, warn user |
| **IndexedDB Quota** | Storage limit exceeded | Clear old cache, prompt user |
