## Validation Rules

### SSCC Generation
- **GS1 Prefix**: Required, 7-10 digits configured in org settings
- **Sequence**: Must be unique per org, auto-incremented
- **Format**: Must be exactly 18 digits (00 + 16 checksum digits)
- **Check Digit**: Calculated per GS1 algorithm (mod 10)
- **Uniqueness**: SSCC must be globally unique (not reused)

### Label Printing
- **Printer**: Required, must be online/reachable
- **Format**: Must be one of [4x6", 4x8"]
- **Copies**: Min 1, max 99
- **Orientation**: Portrait or Landscape
- **Print Mode**: ZPL for Zebra, PDF for others
- **Box Dimensions**: Width >= 10cm, Height >= 10cm, Weight > 0kg, Max dimensions: 200cm per axis (carrier limit for oversized packages)

### Reprint
- **Reason**: Must be selected from: damaged, lost, quality_issue, other
- **Reason Notes**: Optional, max 500 characters
- **Audit Trail**: All reprints logged with user, timestamp, reason

### Batch Operations
- **Batch Size**: Max 100 labels per batch
- **Status Filter**: Cannot print "Error" status until regenerated
- **Same Printer**: All labels in batch must use same printer (or allow per-label override)

## Business Rules

### SSCC Generation
- **Automatic**: Generated when box is sealed/completed in packing station
- **Manual**: Can be manually regenerated if needed
- **Unique**: Each SSCC is globally unique within org
- **Immutable**: Once generated, SSCC cannot be changed (for traceability)
- **Sequence**: Stored in organizations.next_sscc_sequence, incremented per generation
- **Format**: Always GS1-18 format: 00 + company_prefix + serial_number + check_digit

### Label Printing
- **Printer Selection**: Default from org settings, can be overridden per print job
- **Format Selection**: 4x6" for cartons, 4x8" for pallets
- **Native ZPL**: Preferred for Zebra printers (faster, better quality)
- **PDF Fallback**: Used for non-Zebra printers or universal compatibility
- **Auto-Print**: If enabled, skips confirmation dialog and prints immediately
- **Print Queue**: Printed labels added to printer queue, tracked for audit

### Batch Printing
- **Consolidation**: Multiple labels printed in single batch job
- **Status Tracking**: Track which labels printed successfully vs. failed
- **Retry Failed**: Failed labels can be regenerated and reprinted
- **Order**: Can print in order (sequential) or all at once
- **Pause Between**: Optional pause after each label for user intervention
- **Cancel Batch**: User can stop batch operation - shows summary of printed/not printed

### Reprint Policy
- **Audit Trail**: Every print recorded with user, timestamp, printer
- **Reprint Reason**: Required (damaged, lost, quality_issue, other)
- **Reprint Limit**: No hard limit, but tracked for quality analysis
- **Reprint Count**: Incremented for each reprint (max 99 theoretically)
- **Shipment Impact**: Reprint does not affect shipment status or tracking

### SSCC Validation
- **Format**: Must be valid GS1-18 format (18 digits)
- **Check Digit**: Automatically calculated and validated
- **Uniqueness**: No duplicate SSCCs in org
- **Company Prefix**: Must match org's GS1 Company Prefix
- **Audit**: All SSCC validations logged

## GS1 Compliance

### SSCC-18 Format
**Structure**: 00 (extension) + 1234567890 (company prefix) + 123456 (serial) + 8 (check digit)

**Breaking Down Example**: `00 1234 5678 9012 3456 78`
- **00**: Extension digit (always 0 for logistics units)
- **1234567890**: GS1 Company Prefix (7-10 digits, org-specific)
- **123456**: Serial Reference Number (6-8 digits, unique sequentially)
- **78**: Check Digit (calculated per GS1 mod-10 algorithm)

### Barcode Standards
- **Barcode Type**: GS1-128 (Code-128 with GS1 semantics)
- **Data Format**: FNC1 prefix + AI (00) + SSCC value
- **Minimum Size**: 40mm height for reliable scanning
- **Printing**: Minimum 203 DPI (Zebra default)
- **Quiet Zone**: 2.83mm on all sides

### Label Content Requirements (Per GS1)
- **SSCC Barcode**: GS1-128 encoded barcode
- **Human-Readable**: SSCC in format "00 1234 5678 9012 3456 78"
- **Packaging Level**: Information about contents (products, quantities)
- **Shipment Info**: Ship-to address, order number
- **Other Data**: Weight, dimensions, handling instructions

### Traceability
- **Immutable SSCC**: Once assigned, cannot be changed
- **Audit Trail**: All SSCC generations logged
- **Print History**: Track all label prints and reprints
- **Genealogy**: Link SSCC â†’ Shipment â†’ Sales Order â†’ Products â†’ Lots
