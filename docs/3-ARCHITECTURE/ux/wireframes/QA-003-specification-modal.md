## Validation Rules

### Step 1: Basic Information (On Next)
1. **Product**: Required, must exist
2. **Effective Date**: Required, must be today or future date
3. **Expiry Date**: Optional, but if provided must be > effective_date
4. **Status**: Defaults to Draft, immutable until approved
5. **Reviewers**: Optional, but recommended

### Step 2: Parameters (On Next)
1. **At least 1 parameter required**
2. For each Measurement parameter:
   - Target: Required, numeric
   - Min: Required, must be < Target
   - Max: Required, must be > Target
   - Unit: Required, must be valid
3. For each Visual/Attribute parameter:
   - Target: Required, non-empty string
   - Unit: N/A (disabled)
   - Min/Max: N/A (disabled)
4. Test Method: Optional but recommended
5. Critical Flag: Optional, self-documented

### Step 3: Review (On Submit)
1. All Step 1 validations passed
2. All Step 2 validations passed
3. At least 1 parameter defined with valid data
4. No duplicate parameter names within spec
5. Date logic: effective_date <= (expiry_date if provided)

### Edit Mode (Draft Only)
1. Cannot edit product (immutable)
2. Cannot edit version (read-only)
3. Cannot change status from Draft until approved
4. All other fields editable until approved
5. Once approved, spec becomes read-only (must create new version)

## Business Rules

1. **Product Immutability**: Once created, product cannot be changed (prevents spec misassignment)
2. **Version Auto-Increment**: Each new spec auto-increments (v1, v2, v3, etc.)
3. **Draft-Only Editing**: Only draft specs can be edited; approved specs are immutable
4. **Status Workflow**: One-way progression: Draft â†’ PendingApproval â†’ Approved â†’ Archived

7. **Effective/Expiry Logic**: effective_date <= expiry_date (if provided)
8. **Approval Required**: Draft specs must be approved before becoming Active
9. **Previous Version Import**: Can import parameters from approved versions of same product
10. **Template Library**: Org-wide reusable parameter sets for common spec types
11. **No Backtracking**: Once approved, cannot revert to draft (must create new version)
12. **Bulk Import**: Can import multiple parameters at once from template or previous version
