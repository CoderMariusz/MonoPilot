# Story 6.1: LP QA Status Management

## Goal
Enable QC users to assign and track QA status on every LP.

## Acceptance Criteria
- LP has qa_status: pending, passed, failed, quarantine
- QC user can change status
- Timestamp and user recorded
- Audit trail entry created

## Technical
- FR: QC-FR-01, QC-FR-02
- API: PUT /api/warehouse/license-plates/:id/qa-status
- Prerequisites: Epic 5 (LP)
