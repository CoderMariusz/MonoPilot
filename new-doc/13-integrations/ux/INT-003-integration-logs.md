2. **Search Bar** - Text search (event type, integration name, error message)
3. **Filter Controls** - Integration dropdown (All, or specific integration), Status dropdown (All, Success, Warning, Error, Retry), Time Range dropdown (Last 1h, 24h, 7d, 30d, Custom), Event Type dropdown (All, Sync, Export, Import, Webhook, Auth)
4. **Auto-refresh Toggle** - Checkbox to enable/disable auto-refresh (every 30s)
5. **Export Logs Button** - Dropdown (CSV, JSON, XLSX), exports filtered logs

8. **Pagination** - Shows total count, page numbers, next/prev buttons

## Main Actions

### Primary
- **[Export Logs]** - Dropdown to select format (CSV, JSON, XLSX) â†’ exports filtered logs to file
- **[View Details]** - Opens modal with full log data (request, response, metadata, error trace)

### Secondary (Modal Actions)
- **View Related Logs** - Filters logs by same request_id or integration
- **Retry This Request** - Retries failed request (opens confirmation)
- **Copy Request ID** - Copies request ID to clipboard (for support)
- **Copy Payload/Response** - Copies JSON to clipboard

### Filters/Search
- **Search** - Real-time filter by event type, integration, error message
- **Filter by Integration** - Dropdown (All, or specific integration)
- **Filter by Status** - All, Success, Warning, Error, Retry
- **Filter by Time Range** - Last 1h, 24h, 7d, 30d, Custom date range
- **Filter by Event Type** - All, Sync, Export, Import, Webhook, Auth, API Call

## States

- **Loading**: Skeleton rows (8), "Loading logs..." text
- **Empty**: "No logs found" message, "Clear Filters" button
- **Error**: "Failed to load logs" warning, error code, Retry + Contact Support
- **Success**: Table with log rows, filter controls, pagination
- **Auto-refresh Active**: Visual indicator (spinning icon) when enabled

## Technical Notes

- **RLS**: Logs filtered by `org_id`
- **API**:
  - `GET /api/integrations/logs?integration={id}&status={status}&time_range={range}&event_type={type}&search={query}&page={N}&limit={L}`
  - `GET /api/integrations/logs/{id}` (single log details)
  - `POST /api/integrations/logs/{id}/retry` (retry failed request)
  - `POST /api/integrations/logs/export?format={csv|json|xlsx}` (export filtered logs)
- **Real-time**: Subscribe to new logs via Supabase Realtime (if auto-refresh enabled)
- **Pagination**: 20 logs per page
- **Auto-refresh**: Polls every 30s when enabled, auto-disables after 1 hour
- **Retention**: Logs kept for 90 days (configurable per org)
