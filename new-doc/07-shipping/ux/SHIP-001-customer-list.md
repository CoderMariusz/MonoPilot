### 2. KPI Summary Cards (4 cards at top)

**Card 1: Total Customers**
- Value: 42
- API: GET /api/shipping/customers/kpis?metric=total_count
- Refresh: 60 sec (real-time via WebSocket)
- Color: Primary blue (#3B82F6)

**Card 2: Active Customers**
- Value: 38
- Formula: COUNT(customers WHERE is_active = true)
- API: GET /api/shipping/customers/kpis?metric=active_count
- Badge: Green (#10B981)
- Refresh: 60 sec (real-time via WebSocket)

**Card 3: Inactive Customers**
- Value: 4
- Formula: COUNT(customers WHERE is_active = false)
- API: GET /api/shipping/customers/kpis?metric=inactive_count
- Badge: Gray (#6B7280)
- Refresh: 60 sec (real-time via WebSocket)

- Value: 3

- Badge: Amber (#F59E0B)
- Refresh: 60 sec (real-time via WebSocket)

## Business Rules

### Customer Status
- **Active (ğŸŸ¢)**: is_active = true, can be used in sales orders
- **Inactive (ğŸŸ¡)**: is_active = false, cannot create new sales orders, but historical data preserved
- **Soft Delete**: is_active = false, customer_deleted_at = timestamp (data audit trail)

### Customer Code
- **Format**: Auto-generated (CUST-{6-digit sequence}) or manual entry
- **Uniqueness**: Unique per organization
- **Usage**: Reference in sales orders, invoices, shipping labels

### Payment Terms
- **Valid Values**: 7, 15, 30, 45, 60, 90 days
- **Default**: 30 days
- **Usage**: Invoice due date calculation (order_date + payment_terms_days)

### Allergen Restrictions
- **Format**: Array of allergen IDs (from allergens table in Technical module)
- **Usage**: Validation on sales order creation (alert if customer has allergen restriction but order contains allergen)
- **Examples**: "peanut", "tree_nut", "milk", "gluten", "sesame", "sulfites"

### Credit Limits
- **Optional**: Can be null (unlimited credit)
- **Calculation**: credit_limit (max) vs. credit_used (sum of unpaid invoices)
- **Status**:
  - Within Limit: credit_used < credit_limit
  - At Risk: 80% <= (credit_used / credit_limit) < 100%
  - Exceeded: credit_used >= credit_limit

### Search Behavior
- **Case-Insensitive**: Search is case-insensitive
- **Partial Matching**: Match anywhere in field (not just start)
- **Multiple Fields**: OR logic (match any field)
- **Real-time**: Debounced 300ms

### Filtering Behavior
- **AND Logic**: Multiple filters are combined with AND
  - Example: Status = Active AND Category = Wholesale
- **Default**: No filters applied (show all)
- **Persistence**: Filters NOT persisted (reset on page reload)

### Sorting
- **Default**: Sort by name (A-Z)
- **Sortable Columns**: Name, Code, Created Date, Last Order Date
- **Persistence**: Sort order NOT persisted

### Pagination
- **Default Page Size**: 15 customers per page
- **Options**: 15, 25, 50 per page
- **Max Offset**: (total - 1)
- **Empty Page**: If offset > total, return empty results
