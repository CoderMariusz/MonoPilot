## Main Actions

### Scanner Actions

| Action | Trigger | Result |
|--------|---------|--------|
| **Scan Location Barcode** | External scanner or camera | Validate location, display pick lines for that location |
| **Scan License Plate Barcode** | External scanner or camera | Validate product match, FIFO/FEFO check, show quantity form |
| **Adjust Quantity** | +/- buttons or manual entry | Update qty_to_pick, validate max available |
| **Confirm Pick** | Button click | Record pick, update pick_list_lines.quantity_picked, show next |
| **Short Pick** | Button click | Open short pick modal, capture reason, create backorder |
| **Acknowledge Allergen** | Checkbox + button | Log allergen override, proceed with pick |
| **Navigation Map** | Button click | Show aisle/location map, help picker navigate |

### Form Actions

| Action | Result |
|--------|--------|

## Validation Rules

### Pick Confirmation Validation

| Rule | Condition | Action |
|------|-----------|--------|
| **Location Match** | scanned_location != expected_location | Block with error "Wrong location" |
| **Product Match** | lp.product_id != expected_product_id | Block with error "Wrong product" |
| **FIFO Compliance** | lp.mfg_date > oldest_available.mfg_date | Warn "Not picking oldest stock" (allow with confirmation) |
| **FEFO Compliance** | lp.bbd_date > earliest_expiry.bbd_date | Warn "Not picking earliest expiry" (allow with confirmation) |
| **Quantity Valid** | qty_picked > lp.available_qty | Block with error "Exceeds available quantity" |
| **Quantity Zero** | qty_picked = 0 | Block with error "Quantity must be > 0" |
| **Allergen Check** | product.allergens & customer.allergen_restrictions | Show allergen alert modal (require acknowledgment) |

### Short Pick Validation

| Rule | Condition | Action |
|------|-----------|--------|
| **Reason Required** | short_reason is null | Block with error "Reason required" |
| **Qty Available Required** | qty_available is null | Block with error "Quantity available required" |
| **Qty < Requested** | qty_available < qty_to_pick | Allow (this is short pick scenario) |
| **Qty > Ordered** | qty_available > qty_to_pick | Block with error "Cannot exceed ordered quantity" |

### Barcode Validation

| Rule | Condition | Action |
|------|-----------|--------|
| **Location Format** | Not recognized format | Error "Invalid location barcode" |
| **LP Format** | Not recognized LP format | Error "Invalid LP barcode" |
| **Duplicate Scan** | Same barcode scanned twice in 2s | Ignore duplicate |
| **Barcode Timeout** | No scan within 30s | Return to scan prompt |

## Business Rules

### Scanner Workflow

1. **Login Required**: User must authenticate before accessing scanner pick
2. **Active Pick List Required**: Must have assigned pick list with status IN ('pending', 'assigned', 'in_progress')
3. **Warehouse Assignment**: Only show pick lists for user's assigned warehouse
4. **Location Sequence**: Must pick locations in sequence (optimized route)
5. **FIFO Default**: Apply FIFO by default for stable products
6. **FEFO Default**: Apply FEFO for perishable products (marked in product settings)
7. **Allergen Enforcement**: Show alert for customer allergen restrictions, require acknowledgment
8. **Location Validation**: Scan location barcode before showing pick lines
9. **LP Validation**: Scan LP to validate product + location match
10. **Offline Support**: Queue operations in IndexedDB (max 100), sync when online
11. **Session Timeout**: Auto-logout after warehouse_settings.scanner_idle_timeout_sec (default 300s)
12. **Pick Sequence**: System determines optimal pick order per location

### Feedback Rules

| Event | Audio | Vibration | Visual |
|-------|-------|-----------|--------|
| **Successful Scan** | Success beep (high tone) | Short pulse (100ms) | Green flash |
| **Invalid Scan** | Error beep (low tone) | Double pulse (100ms x 2) | Red flash |
| **Pick Confirmed** | Success chime (multi-tone) | Success pattern (200ms) | Green checkmark |
| **FIFO/FEFO Warning** | Warning beep (mid tone) | Long pulse (300ms) | Yellow warning |
| **Allergen Alert** | Alert tone | Triple pulse | Red header |
| **Offline Mode** | Alert tone | Triple pulse | Orange banner |

### Offline Mode Rules

1. **Storage**: IndexedDB for offline queue (max 100 operations)
2. **Validation**: Local validation only (no server checks)
3. **Queue Order**: FIFO (First-In, First-Out)
4. **Auto-Sync**: When connection restored, sync queue in order
5. **Conflict Resolution**: Server-side validation on sync, flag conflicts
6. **Storage Limit**: Warn at 80% (80 operations), block at 100
7. **TTL**: Queued operations expire after 24h (warn user)

6. **Background Sync**: Sync offline queue using Background Sync API

## Error Handling

### Network Errors

| Error | Trigger | Action |
|-------|---------|--------|
| **Connection Lost** | Network offline | Switch to offline mode, queue operations |
| **Timeout (>5s)** | No response from server | Retry once, then offline mode |
| **500 Server Error** | Server failure | Show error, retry button, offline fallback |
| **401 Unauthorized** | Session expired | Force logout, redirect to login |
| **403 Forbidden** | Permission denied | Show error "No permission for this warehouse" |

### Validation Errors

| Error | Trigger | User Action |
|-------|---------|-------------|
| **LOCATION_MISMATCH** | Location barcode != expected | Scan correct location or navigate manually |
| **PRODUCT_MISMATCH** | LP product != expected | Scan correct LP or report inventory issue |
| **FIFO_VIOLATION** | Picking newer than oldest | Proceed with caution (warn) or pick from oldest |
| **QUANTITY_EXCEEDED** | qty > available | Reduce quantity or trigger short pick |
| **ALLERGEN_ALERT** | Customer has restriction | Acknowledge and separate handling |

### Scan Errors

| Error | Trigger | Recovery |
|-------|---------|----------|
| **Camera Permission Denied** | User denied camera access | Prompt to enable in settings |
| **Scanner Hardware Failure** | External scanner disconnected | Show error, prompt to reconnect |
| **Barcode Unreadable** | Poor quality barcode | Retry scan, suggest manual entry |
| **Duplicate Scan** | Same barcode scanned twice in 2s | Ignore, no action |
| **Scan Timeout** | No scan within 30s | Return to scan prompt, clear state |

### Offline Queue Errors

| Error | Trigger | Recovery |
|-------|---------|----------|
| **Queue Full** | 100 operations in queue | Block new picks, prompt to sync |
| **Sync Conflict** | Server validation fails on sync | Flag operation, show conflict resolution UI |
| **TTL Expired** | Queued operation >24h old | Remove from queue, warn user |
| **IndexedDB Quota** | Storage limit exceeded | Clear old cache, prompt user |
