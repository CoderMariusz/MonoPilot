## Business Rules

### Cost Simulation
- **Real-time Recalculation**: Cost updates within 200ms of any override change
- **Material Override**: Override price replaces standard price for that item only
- **Yield Override**: Affects labor and overhead per unit: new_cost = standard_cost / (yield_percent / 100)
  - Example: 105% yield â†’ labor cost = $0.40 / 1.05 = $0.38 per unit (5% reduction)
- **Overhead Rate Override**: Replaces standard overhead rate for simulation
- **Allergen Detection**: Automatically detect allergen changes when material substitutions change allergen profile

### BOM Version Comparison
- **Change Type Detection**:
  - Added: Item exists in new version but not in previous
  - Removed: Item exists in previous version but not in new
  - Modified: Item exists in both but qty/price changed
  - Unchanged: Item exists in both with same qty/price
- **Baseline Version**: First selected version is baseline for delta calculation
- **Cost Trend**: Show cost evolution across versions chronologically

### Save as New BOM Version
- **Version Numbering**: Auto-increment (e.g., v1.2 â†’ v1.3 or v2.0)
- **Effective Date**: Default to today, user can override
- **Status**: New version defaults to "draft", must be activated manually
- **Standard Cost Update**: Create new standard_costs records for new version

### Query Optimization
- **Simulation Calculation**: In-memory calculation (no DB queries), <200ms response
- **BOM Version Comparison**: Single query with JOINs (boms, bom_items, products), index on (org_id, product_id, version)
- **Allergen Detection**: Join with products_allergens table, cache for 5 min
