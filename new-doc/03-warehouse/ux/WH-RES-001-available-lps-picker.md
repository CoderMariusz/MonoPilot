## Business Rules

1. **FIFO Default**: Oldest LP (created_at ASC) highlighted first
2. **FEFO Override**: If enabled, soonest expiry (expiry_date ASC) highlighted
3. **NULL Expiry Last**: LPs without expiry sort last in FEFO
4. **QA Filter**: Only qa_status='passed' shown
5. **Expired Excluded**: expiry_date < today excluded
6. **Auto-select Logic**: Selects top N LPs (by suggestion order) to meet required_qty
7. **Partial LP**: User can input qty < full LP qty
8. **Violation Warning**: Shows when non-optimal LP selected (non-blocking)
9. **Multi-LP**: Supports selecting multiple LPs (100kg = 40+50+10)
10. **Available Calculation**: Shows qty - reserved_qty for each LP

## Technical Notes

- **Service Call**: `findAvailableLPs(productId, warehouseId, strategy)`
- **Real-time Qty**: Fetch available_qty on modal open
- **Strategy Toggle**: Re-sort table when strategy changed
- **Validation**: Sum of selected_qty must not exceed required_qty (can be less)
- **Optimistic Update**: Show selected qty immediately, validate on Reserve click
