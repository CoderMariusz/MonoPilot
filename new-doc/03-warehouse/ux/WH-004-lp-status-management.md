## Overview

This wireframe covers the **LP Status Management infrastructure** that supports Epic 04 Production and other warehouse operations:

**Components Covered**:
1. Status Badges (LP Status + QA Status)
2. Change LP Status Modal (status transitions)
3. Block LP Modal (with reason)
4. Unblock LP Modal (confirmation)
5. Status Audit Trail (change history display)

**Out of Scope** (covered by other wireframes):
- QA Status Change Modal â†’ See WH-009
- LP List filtering by status â†’ Story 05.5
- Scanner UI for status changes â†’ Phase 2

## Business Rules

### Status Transition Matrix

| Current Status | New Status | Valid? | Reason Required? | Side Effects |
|----------------|------------|--------|------------------|--------------|
| available â†’ reserved | Yes | No | None |
| available â†’ consumed | Yes | Yes (min 10 chars) | Terminal state warning |
| available â†’ blocked | Yes | Yes (min 10 chars) | Consumption/movement blocked |
| reserved â†’ available | Yes | No | Release reservation |
| reserved â†’ consumed | Yes | Yes | Terminal state warning |
| reserved â†’ blocked | No | N/A | Must release first |
| consumed â†’ * | No | N/A | Terminal state (no transitions out) |
| blocked â†’ available | Yes | Yes (min 10 chars) | Unblock |
| blocked â†’ consumed | No | N/A | Must unblock first |

### Consumption Validation Rules

**Consumption Allowed** when:
- `status IN ('available', 'reserved')` AND
- `qa_status = 'passed'`

**Consumption Blocked** when:
- `status = 'consumed'` â†’ Error: "LP already consumed"
- `status = 'blocked'` â†’ Error: "LP is blocked"
- `qa_status != 'passed'` â†’ Error: "LP not QA approved (qa_status: {value})"

### Block/Unblock Rules

**Can Block**:
- Status: `available`, `consumed` (but not useful)
- Cannot block: `reserved` (must release reservation first)
- Reason required: Min 10 characters

**Can Unblock**:
- Only from `status = 'blocked'`
- Reason required: Min 10 characters
- Returns to: `available`

**Side Effects**:
- Block: All operations disabled (consumption, movement, split/merge)
- Unblock: All operations enabled (if QA passed)

### Audit Trail Rules

**Mandatory Recording**:
- Every status change (status or qa_status)
- Every block/unblock action
- Every location change (via stock_moves)
- LP creation event

**Audit Entry Fields**:
- `field_name`: 'status' | 'qa_status' | 'location_id'
- `old_value`: Previous value (null for creation)
- `new_value`: Updated value
- `reason`: User-provided reason (optional for some transitions)
- `changed_by`: User ID (or 'system' for automated changes)
- `changed_at`: Timestamp
- `metadata`: JSON with additional context (action_type, notifications, etc.)

**Retention**: 2 years minimum, 7 years recommended
