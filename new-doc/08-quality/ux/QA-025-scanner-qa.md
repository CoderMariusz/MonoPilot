### Inspection Validation

| Rule | Condition | Action |
|------|-----------|--------|
| **Product required** | No product/LP scanned | Block "Product required" |
| **Result required** | No Pass/Fail selected | Block "Select Pass or Fail" |
| **Parameter required** | Parameter marked required & null | Block with field error |

| **Voice note** | If voice_required=true & null | Block "Voice note required" |

| Rule | Condition | Action |
|------|-----------|--------|
| **Value out of range** | value < LLC OR value > ULC | Trigger full-screen deviation alert + NCR auto-creation |
| **Supervisor approval** | Deviation detected | Require supervisor PIN/biometric |
| **Value required** | measured_value null | Block "Value required" |
| **Unit mismatch** | Value scale incorrect | Warn "Check unit (Â°C vs Â°F)" |

| Rule | Condition | Action |

## Business Rules

### Scanner QA Workflow

1. **Warehouse Assignment**: Only show operations for user's assigned warehouse
2. **Task Filtering**: Show pending inspections + scheduled CCPs
3. **Auto-Population**: Scan product/LP â†’ auto-fill form with details
4. **CCP Deviation Alert**: Out-of-range values trigger full-screen alert + supervisor notification
5. **NCR Auto-Creation**: Repeated failures or threshold breaches â†’ auto-create NCR (PRD Line 426)
6. **Work Order Context**: CCP/checkpoints linked to WO operations include work_order_id + operation_id
7. **Operation Checkpoints** (FR-QA-026): Operation-specific quality metrics distinct from generic checkpoints
8. **Offline Support**: Queue up to 50 QA operations in IndexedDB
9. **Audit Trail**: Every inspection/CCP/checkpoint creates immutable record
10. **Photo Compression**: Auto-compress images before upload (max 2MB)
11. **Voice Encoding**: Auto-encode voice notes to MP3 (max 2 min)
12. **LP QA Status Update**: Pass â†’ approved, Fail â†’ rejected, Review â†’ pending
13. **Session Timeout**: Auto-logout after warehouse_settings.scanner_idle_timeout_sec

### Feedback Rules

| Event | Audio | Vibration | Visual |
|-------|-------|-----------|--------|
| **Successful Scan** | Success beep | Short pulse (100ms) | Green flash |
| **Invalid Scan** | Error beep | Double pulse | Red flash |
| **Inspection Submitted** | Success chime | Success pattern | Green checkmark |
| **CCP In Range** | Confirmation beep | Short pulse | Green indicator |
| **CCP Out of Range** | Alarm beep (repeating) | Strong pulse (500ms) | Red full-screen + NCR indicator |
| **Offline Mode** | Alert tone | Triple pulse | Orange banner |

### Offline Mode Rules

1. **Storage**: IndexedDB (max 50 QA operations, 24h TTL)
2. **Validation**: Local validation only
3. **Queue Order**: FIFO
4. **Auto-Sync**: When connection restored
5. **Media Handling**: Attach photos/voice after reconnection
6. **Conflict**: Flag operations that fail server-side validation

| **401 Unauthorized** | Session expired | Force logout |
| **500 Server error** | Server failure | Show error, retry, offline fallback |

### Offline Queue Errors

| Error | Trigger | Action |
|-------|---------|--------|
