## Main Actions

### Primary
- **KPI Card Click** - Navigate to respective detail page (pending inspections, yield details, NCR list, CCP monitoring, holds list, CoA list, backlog list, supplier ratings)
- **Auto-Refresh Toggle** - Enable/disable automatic data refresh
- **Manual Refresh Button** - Trigger immediate dashboard data refresh

### Secondary (Alert Panel)
- **View Critical CCP** - Navigate to CCP monitoring detail
- **Inspect Now** - Open inspection workflow for overdue item
- **Release/Reject Hold** - Open hold resolution workflow
- **Review Failed Inspection** - Navigate to inspection detail

### Tab Actions
- **Switch Tab** - Change view between Inspections, NCRs, HACCP Monitoring
- **View Inspection Details** - Navigate to inspection detail page
- **Apply Filters** - Filter inspections by type/status/date/inspector

### Filters
- **Type Filter**: Dropdown (All, Incoming, In-Process, Final)
- **Status Filter**: Dropdown (All, Pending, Passed, Failed)
- **Date Range Filter**: Date picker (Last 7 days, Last 30 days, Custom)
- **Inspector Filter**: Dropdown (All, or inspector list)

### Export
- **Export Quality Report** - Downloads dashboard data to PDF/CSV

## States

- **Loading**: Skeleton cards (8 KPI cards) + skeleton alert panels + skeleton table, "Loading quality dashboard data..." text
- **Success**: Populated 8 KPI cards, alert panels (if alerts exist), recent inspections table, quality trends chart, auto-refresh active
- **Empty**: "No quality data yet" message, Quick actions to create first inspection / set up specs / configure HACCP, Quality Guide link
- **Error**: "Failed to load quality dashboard" warning, Retry + Contact Support buttons, Quick actions still available, Auto-refresh disabled

## Business Rules

### KPI Cards (8 Cards)
- **Pending Inspections**: COUNT(*) WHERE status='pending'
  - Highlight overdue count: WHERE scheduled_date < today()
  - Show yellow indicator (ğŸŸ¡) if overdue_count > 0
- **First Pass Yield**: (COUNT passed / COUNT total) Ã— 100
  - Target: >95%
  - Status: Green (>95%), Yellow (90-95%), Red (<90%)
- **NCR Rate**: (COUNT NCRs / COUNT batches) Ã— 1000
  - Target: <5 per 1000 batches
  - Status: Green (<5), Yellow (5-10), Red (>10)
- **CCP Compliance**: (COUNT within_limits / COUNT total) Ã— 100
  - Target: >99%
  - Status: Green (>99%), Yellow (98-99%), Red (<98%)
- **Holds Active**: COUNT(*) WHERE status='active'
  - Breakdown by severity: critical, major, minor
  - Show count by severity
- **CoA Pending**: COUNT(*) WHERE status='pending'
  - Calculate avg turnaround: AVG(issue_date - created_at)
  - Target: <24h per CoA
- **Inspection Backlog**: COUNT(*) WHERE status='pending'
  - Calculate avg age: AVG(today() - scheduled_date)
  - Status: Green (<1d), Yellow (1-2d), Red (>2d)
- **Supplier Quality Score**: Weighted AVG(overall_score)
  - Target: >85
  - Status: Green (>85), Yellow (75-85), Red (<75)

### Critical Quality Alerts
- **Critical CCPs**:
  - Out of spec: measured_value outside (critical_limit_min, critical_limit_max)
  - Equipment malfunction: manual deviation entry
  - Auto-create deviation record
  - Alert QA Manager + Production Lead (email + SMS)
- **Overdue Inspections**:
  - scheduled_date < today() AND status='pending'
  - Sort by days overdue DESC
  - Show top 3, link to view all
- **Aging Holds**:
  - status='active' AND held_at < today() - 48 hours
  - Sort by hold duration DESC
  - Show top 2, link to view all
- **Failed Inspections Today**:
  - result='failed' AND inspection_date >= today()
  - Show failed parameter, spec, measured value
  - Link to create NCR if not exists

### Recent Inspections Tab
- **Default View**: Last 10 inspections, ordered by inspection_date DESC
- **Filters**: AND logic for multiple filters
  - Type: incoming, in-process, final
  - Status: pending, passed, failed
  - Date Range: custom start/end dates
  - Inspector: filter by inspector_id
- **Pagination**: 10 per page (desktop/tablet), 5 per page (mobile)

### Quality Trends
- **Time Range**: Last 30 days (4 weeks)
- **Aggregation**: Weekly average
- **Comparison**: Current 30 days vs. previous 30 days
- **Trend Indicators**: â†‘ (up), â†“ (down), â†’ (stable)

### Auto-Refresh
- **Default Interval**: 30 seconds (from quality_settings.dashboard_refresh_seconds)
- **Min Interval**: 5 seconds (prevent server overload)
- **Max Interval**: 300 seconds (5 minutes)
- **Toggle**: User can disable auto-refresh (persists in local storage)
- **Manual Refresh**: Always available, bypasses auto-refresh timer
- **Refresh Scope**: KPIs, Alerts, Recent Inspections, Trends (all dashboard data)
- **Error Handling**: If refresh fails, retry after interval, show error toast (non-blocking)

## Error Handling

### API Errors
- **KPIs Fetch Failed**: Show error state in KPI section, rest of dashboard still works
- **Alerts Fetch Failed**: Show error in alert panel section, rest of dashboard still works
- **Inspections Fetch Failed**: Show error in inspection table section, rest of dashboard still works
- **Trends Fetch Failed**: Show error in trends section, rest of dashboard still works

### Partial Failures
- **If KPIs fail**: Show error message in KPI section, Alerts and Inspections and Trends load normally
- **If Alerts fail**: Show error in alert panel section, KPIs and Inspections and Trends load normally
- **If Inspections fail**: Show error in inspection table section, KPIs and Alerts and Trends load normally
- **If Trends fail**: Show error in trends section, KPIs and Alerts and Inspections load normally
- **Each section has independent error boundary**: Errors don't cascade

### Network Timeout
- **KPIs**: 5s timeout, retry once on failure
- **Alerts**: 5s timeout, retry once on failure
- **Inspections**: 5s timeout, retry once on failure
- **Trends**: 5s timeout, retry once on failure
- **Auto-Refresh**: If refresh fails, show toast notification, retry after next interval

### Auto-Refresh Errors
- **Error Handling**: Non-blocking error toast, auto-refresh continues after interval
- **Retry Logic**: Exponential backoff if multiple failures (30s â†’ 60s â†’ 120s)
- **Disable on Repeated Failure**: After 3 consecutive failures, disable auto-refresh, show error banner with "Retry" button
