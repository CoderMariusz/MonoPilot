M05 — Scanner Workflows

Scanner Workflows (6)

  • 📦 Receive: Scan PO → Scan Product → Enter Qty → Confirm → Print Label
  • 🔄 Move: Scan LP → Scan Location → Confirm
  • 🔍 Putaway: Scan LP → System Suggests → Scan Location → Confirm
  • ⚙ Pick: Scan WO → System Suggests LP (FIFO/FEFO) → Scan LP → Confirm
  • ✂ Split: Scan LP → Enter Qty → Confirm → Print Label
  • 🔃 Merge: Scan Primary → Scan Additional → Confirm

Scanner UX Requirements

  • 📱 Touch targets ≥48px
  • 🔊 Audio/vibration feedback
  • 🔑 Username + PIN login
  • 🔒 LP locking (concurrent access protection)
  • ⚡ Scan-first input (auto-focus barcode field)
  • 🔄 Liniowy flow krok po kroku
  • 🔌 Offline queue Phase 2 (IndexedDB, max 100 transakcji)

💡 Dedykowane strony /scanner/* — optymalizowane dla skanerów mobilnych (Zebra TC21/TC26, Honeywell CT40/CT60)

Kluczowe Decyzje Architektoniczne

ADR Decyzja Uzasadnienie
ADR-001 License Plate Inventory Atomowa jednostka inwentarza, NO loose qty. Pełna trasowalność przez lp_genealogy (split/merge/consume/output)
ADR-002 BOM Snapshot WO kopiuje BOM przy tworzeniu. Niezmienny w trakcie produkcji. Zapobiega błędom zmian BOM mid-production
ADR-003 Multi-Tenancy RLS org_id UUID NOT NULL na WSZYSTKICH tabelach. RLS policy: USING (org_id = auth session). Zero cross-org data leaks
ADR-004 GS1 Compliance GTIN-14 produkty, GS1-128 LP (lot/expiry), SSCC-18 palety. AI(01) GTIN, AI(10) Lot, AI(17) Expiry. Standard branżowy
ADR-005 FIFO/FEFO FEFO domyślnie (expiry_date ASC), FIFO fallback (created_at ASC). Override z audit trail. Minimalizacja strat
ADR-006 Scanner-First Dedykowane /scanner/* routes, 48px touch targets, scan-first input, liniowy flow. Optymalizacja dla operatorów hali
ADR-007 WO State Machine DRAFT → RELEASED → IN_PROGRESS → COMPLETED (+ CANCELLED). Guards: hasBOM, hasMaterials, outputRecorded
ADR-008 Audit Trail Hybrydowy: PG triggers + app context. Tabela audit_log (old_data/new_data, user_id, ip, reason). FDA 21 CFR Part 11, FSMA 204
